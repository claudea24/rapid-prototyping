SQL

Queries, joins, window functions, and optimization

Query Execution Order Key

SQL doesn't run in the order you write it:

1. FROM / JOIN
2. WHERE
3. GROUP BY
4. HAVING
5. SELECT
6. ORDER BY
7. LIMIT

JOIN Types Key

  • INNER JOIN — matching rows in both tables
  • LEFT JOIN — all left rows + matching right
  • RIGHT JOIN — all right rows + matching left
  • FULL OUTER — all rows from both sides
  • CROSS JOIN — cartesian product
  • Self Join — table joined to itself

Window Functions Key

SELECT
  name, dept, salary,
  RANK() OVER(
    PARTITION BY dept
    ORDER BY salary DESC
  ) AS dept_rank
FROM employees;

Common: ROW_NUMBER, RANK, DENSE_RANK, LAG, LEAD, SUM, AVG

Aggregation Gotchas Watch

  • COUNT(*) counts all rows incl. NULLs
  • COUNT(col) skips NULLs
  • WHERE filters rows, HAVING filters groups
  • Can't use aliases in WHERE (runs before SELECT)
  • GROUP BY — every non-aggregated column in SELECT must be in GROUP BY

CTEs & Subqueries Tip

WITH monthly_revenue AS (
  SELECT
    DATE_TRUNC('month', dt) AS m,
    SUM(amt) AS rev
  FROM orders
  GROUP BY 1
)
SELECT m, rev,
  LAG(rev) OVER(ORDER BY m) AS prev
FROM monthly_revenue;

Performance Tips Tip

  • Use EXPLAIN to see query plan
  • Index columns used in WHERE and JOIN
  • Avoid SELECT * in production
  • Filter early with WHERE before JOIN
  • Use EXISTS over IN for large subqueries

Python

Data structures, Pandas, NumPy, and coding patterns

Data Structures Complexity Key

  • list — append O(1), index O(1), search O(n), insert O(n)
  • dict — get/set O(1) avg, iteration O(n)
  • set — add/remove/lookup O(1) avg
  • deque — append/pop both ends O(1)
  • heap — push/pop O(log n), min O(1)

Pandas Essentials Key

df.groupby('col')['val'].agg(['mean','sum'])
df.merge(df2, on='id', how='left')
df.pivot_table(values='rev',
  index='month', aggfunc='sum')
df.fillna(0) | df.dropna()
df.apply(lambda x: x * 2)

NumPy Patterns Tip

import numpy as np
a = np.array([1,2,3])
np.mean(a) | np.std(a) | np.median(a)
np.dot(a, b)        # dot product
np.linalg.inv(M)   # matrix inverse
a[a > 2]            # boolean indexing
np.random.normal(0, 1, 1000)

Common Coding Patterns Key

  • Two pointers — sorted array problems
  • Hash map — counting, grouping, lookup
  • Sliding window — subarray/substring
  • Sorting + greedy — optimization
  • defaultdict / Counter — frequency counts
from collections import Counter
c = Counter(words)
c.most_common(10)

String & List Tricks Tip

# List comprehension
[x**2 for x in range(10) if x % 2 == 0]

# Zip two lists
dict(zip(keys, values))

# Sort by custom key
sorted(data, key=lambda x: x[1], reverse=True)

Gotchas Watch

  • Mutable defaults — never use def f(x=[])
  • Shallow copy — a = b[:] or copy.deepcopy()
  • Integer division — // truncates toward -inf
  • is vs == — is checks identity, == checks value

Machine Learning

Algorithms, evaluation metrics, and model selection

Supervised Algorithms Key

  • Linear Regression — continuous target, assumes linearity
  • Logistic Regression — binary classification, outputs probabilities
  • Decision Trees — interpretable, prone to overfitting
  • Random Forest — bagging ensemble, reduces variance
  • Gradient Boosting (XGBoost) — sequential trees, reduces bias
  • SVM — finds max-margin hyperplane, kernel trick
  • KNN — instance-based, no training, slow at scale

Unsupervised Algorithms Key

  • K-Means — partition into k clusters by centroid distance
  • Hierarchical — agglomerative / divisive dendrogram
  • DBSCAN — density-based, finds arbitrary-shaped clusters
  • PCA — dimensionality reduction, max variance directions
  • t-SNE / UMAP — non-linear embedding for visualization

Evaluation Metrics Key

  • Accuracy — (TP+TN) / total — misleading if imbalanced
  • Precision — TP / (TP+FP) — "of predicted positive, how many correct?"
  • Recall — TP / (TP+FN) — "of actual positive, how many found?"
  • F1 Score — harmonic mean of precision & recall
  • AUC-ROC — area under TPR vs FPR curve
  • RMSE / MAE — regression error metrics

Bias-Variance Tradeoff Key

Bias = error from wrong assumptions (underfitting).
Variance = error from sensitivity to training data (overfitting).

  • High bias → try more features, complex model
  • High variance → more data, regularization, simpler model
  • L1 (Lasso) → feature selection (sparse)
  • L2 (Ridge) → shrinks coefficients (stable)

Feature Engineering Tip

  • One-hot encoding — categorical → binary columns
  • Label encoding — ordinal categories
  • Scaling — StandardScaler (z-score), MinMaxScaler
  • Log transform — right-skewed distributions
  • Feature crosses — interaction terms
  • Target encoding — category → mean of target (watch leakage!)

Cross-Validation Watch

  • k-Fold CV — split data into k folds, rotate test set
  • Stratified k-Fold — preserves class distribution
  • Time Series CV — expanding or sliding window (no future data!)
  • Data leakage — fit scalers/encoders ONLY on training fold

Statistics

Hypothesis testing, distributions, and experiment design

Descriptive Stats Key

  • Mean — sensitive to outliers
  • Median — robust to outliers, better for skewed data
  • Mode — most frequent value
  • Variance — avg squared deviation from mean
  • Std Dev — sqrt(variance), same units as data
  • IQR — Q3 - Q1, used for outlier detection

Hypothesis Testing Key

  • H0 — null hypothesis (no effect / no difference)
  • H1 — alternative hypothesis
  • p-value — P(data | H0 true). Reject H0 if p < alpha
  • Type I error — false positive (reject true H0)
  • Type II error — false negative (fail to reject false H0)
  • Power = 1 - P(Type II) — probability of detecting real effect

Common Tests Key

  • z-test — known population variance, large n
  • t-test — unknown variance, small n
  • Paired t-test — before/after on same subjects
  • Chi-squared — categorical independence
  • ANOVA — compare means across 3+ groups
  • Mann-Whitney U — non-parametric alternative to t-test

A/B Testing Tip

  • 1. Define metric and hypothesis
  • 2. Calculate sample size (power analysis)
  • 3. Randomize users into control/treatment
  • 4. Run for predetermined duration
  • 5. Analyze with appropriate test
  • Pitfalls: peeking, multiple comparisons, novelty effect, Simpson's paradox

Confidence Intervals Formula

CI = x̄ ± z * (σ / √n)

A 95% CI means: if we repeated sampling, 95% of intervals would contain the true parameter. It does not mean 95% probability the parameter is in this interval.

Central Limit Theorem Key

Sample means approach a normal distribution as n increases, regardless of population distribution (given finite variance). Generally applies when n ≥ 30.

SE = σ / √n

Probability

Rules, distributions, and Bayes' theorem

Fundamentals Key

  • P(A or B) = P(A) + P(B) - P(A and B)
  • P(A and B) = P(A) · P(B|A)
  • Independent: P(A and B) = P(A) · P(B)
  • Complement: P(A') = 1 - P(A)
  • Permutations: n! / (n-r)!
  • Combinations: n! / (r!(n-r)!)

Bayes' Theorem Formula

P(A|B) = P(B|A) · P(A) / P(B)

Prior × Likelihood / Evidence = Posterior

Classic interview Q: "A test is 99% accurate. Disease prevalence is 1%. If positive, what's the probability you have it?" Answer: ~50%, not 99%!

Key Distributions Key

  • Bernoulli — single trial, p success
  • Binomial — n trials, count successes
  • Poisson — count of events in fixed interval (λ)
  • Geometric — trials until first success
  • Normal — bell curve, defined by μ and σ
  • Exponential — time between Poisson events
  • Uniform — equal probability over range

Expected Value & Variance Formula

E[X] = Σ x · P(x)
Var(X) = E[X²] - (E[X])²
  • E[aX + b] = aE[X] + b
  • Var(aX + b) = a² Var(X)
  • If independent: Var(X+Y) = Var(X) + Var(Y)

Law of Large Numbers Key

As sample size grows, the sample mean converges to the population mean. This is why A/B tests work — given enough data, observed averages reflect truth.

Classic Interview Puzzles Tip

  • Birthday problem — 23 people → ~50% chance of shared birthday
  • Monty Hall — always switch (2/3 vs 1/3)
  • Coupon collector — E[n] = n · H(n) for n types
  • Dice problems — use complementary counting or E[X]

Case Studies

Product sense, metrics, and business frameworks

Product Metrics Framework Key

  • North Star Metric — single metric capturing core value
  • Acquisition — new users, sign-ups, installs
  • Activation — first key action (aha moment)
  • Retention — D1, D7, D30 retention rates
  • Revenue — ARPU, LTV, conversion rate
  • Referral — viral coefficient, NPS

Structuring a Case Tip

  1. Clarify — restate the problem, ask questions
  2. Define metrics — primary + guardrail metrics
  3. Hypothesize — state expected cause / direction
  4. Analyze — segment data, test hypotheses
  5. Recommend — action + expected impact
  6. Measure — how to validate the recommendation

Metric Change Questions Key

"Metric X dropped 10%. What happened?"

  • 1. Clarify — which metric, how measured, timeframe?
  • 2. External factors — seasonality, competitors, market
  • 3. Internal factors — product changes, bugs, deploys
  • 4. Segment — by platform, geo, user cohort, channel
  • 5. Quantify — how much does each factor explain?

Market Sizing Tip

Top-down or bottom-up estimation:

  • Top-down: Total market → apply filter rates
  • Bottom-up: Unit economics → scale up
  • State assumptions clearly
  • Use round numbers for mental math
  • Sanity-check your answer

Experiment Design Key

  • Unit of randomization — user, session, page?
  • Duration — cover full weekly cycle minimum
  • Guardrail metrics — ensure you don't break things
  • Network effects — may need cluster randomization
  • Novelty / primacy effects — wait for stabilization

Communication Tips Tip

  • Lead with the answer — conclusion first, then evidence
  • Structure your thoughts — "I'll break this into 3 parts..."
  • Quantify everything — "~20% of users" not "a lot of users"
  • Acknowledge trade-offs — shows maturity
  • Ask clarifying questions — never assume
