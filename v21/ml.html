Machine Learning — Models & Evaluation

Bias-variance tradeoff, evaluation metrics, class imbalance, data leakage, and core algorithms.

1 Bias-Variance Tradeoff

▾

Bias measures how far off predictions are from true values on average. Variance measures how much predictions change with different training sets.

ConditionBiasVarianceProblemFix
UnderfittingHighLowModel too simpleMore features, complex model
OverfittingLowHighModel memorizes noiseRegularization, more data, simplify
Good fitLowLow--
Interview Insight

High bias = underfitting. The model is too simple to capture the pattern. This is the #1 way interviewers phrase this concept.

2 Evaluation Metrics & Class Imbalance

▾

Accuracy is misleading when classes are imbalanced. A fraud detection model with 0.5% fraud rate gets 99.5% accuracy by predicting "not fraud" for everything.

MetricFormulaUse When
Accuracy(TP+TN) / TotalBalanced classes only
PrecisionTP / (TP+FP)Cost of false positives is high (spam filter)
RecallTP / (TP+FN)Cost of false negatives is high (cancer detection)
F1 Score2 * P * R / (P+R)Balance precision and recall
AUC-ROCArea under ROC curveOverall ranking quality
Key Insight

When someone says a model has 99.5% accuracy on fraud detection, your first question should be: "What's the class distribution?" A naive baseline of predicting the majority class would achieve similar accuracy.

3 Data Leakage

▾

Data leakage occurs when information from the test set (or future data) influences the training process, giving unrealistically good performance.

TypeExamplePrevention
Target leakageFeature derived from target variableReview feature provenance carefully
Train-test contaminationNormalizing before splittingAlways split first, then preprocess
Temporal leakageUsing future data to predict pastUse time-based splits
Interview Tip

The correct pipeline order: split → fit scaler on train → transform both. Never fit on the full dataset before splitting.
