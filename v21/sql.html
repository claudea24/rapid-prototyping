SQL — The Language of Data

Master query execution order, joins, window functions, aggregation, CTEs, and optimization.

1 Logical Order of Query Execution

▾

The engine processes clauses in a different logical order than you write them. This explains why you can't use a SELECT alias in WHERE.

1
FROM / JOIN
Identify source tables and perform joins
2
WHERE
Filter rows
3
GROUP BY
Create groups for aggregation
4
HAVING
Filter groups (after aggregation)
5
SELECT
Evaluate expressions, aliases created here
6
ORDER BY
Sort results (can use aliases)
7
LIMIT
Return subset of rows
Interview Insight

This is one of the most commonly asked SQL fundamentals questions. Memorize the order and understand why each step depends on the previous.

2 JOIN Types

▾

Understanding which rows are preserved in each join type is critical.

JoinKeepsUse When
INNER JOINOnly matching rowsYou need rows that exist in both tables
LEFT JOINAll left rows + matchesYou need all records from the primary table
RIGHT JOINAll right rows + matchesRarely used; rewrite as LEFT JOIN
FULL OUTERAll rows from bothFinding mismatches between tables
CROSS JOINCartesian productGenerating all combinations
Common Pitfall

A LEFT JOIN with a WHERE filter on the right table effectively becomes an INNER JOIN. Move the filter to the ON clause instead.

3 Window Functions & Ranking

▾

Window functions compute values across a set of rows related to the current row, without collapsing them like GROUP BY.

FunctionTiesGapsExample (scores 90,90,80)
ROW_NUMBER()Breaks arbitrarilyNo1, 2, 3
RANK()Same rankYes (skips)1, 1, 3
DENSE_RANK()Same rankNo1, 1, 2
SELECT name, score,
  RANK() OVER (ORDER BY score DESC) AS rnk,
  DENSE_RANK() OVER (ORDER BY score DESC) AS d_rnk
FROM students;
Interview Tip

RANK() skips after ties (1,1,3), DENSE_RANK() does not (1,1,2). Know this cold.

4 CTEs & Subqueries

▾

A Common Table Expression (CTE) uses WITH to define a named temporary result set. CTEs improve readability and allow recursion.

WITH monthly_revenue AS (
  SELECT DATE_TRUNC('month', order_date) AS month,
         SUM(amount) AS revenue
  FROM orders
  GROUP BY 1
)
SELECT month, revenue,
  LAG(revenue) OVER (ORDER BY month) AS prev_month
FROM monthly_revenue;
FeatureCTESubquery
ReadabilityNamed, easy to followNested, can get messy
ReusabilityReferenced multiple timesMust duplicate
RecursionSupportedNot supported

5 Aggregation & GROUP BY

▾

GROUP BY collapses rows into groups. Every column in SELECT must either be in GROUP BY or wrapped in an aggregate function.

FunctionPurposeNULL Handling
COUNT(*)Count all rowsCounts NULLs
COUNT(col)Count non-NULL valuesIgnores NULLs
SUM(col)Sum valuesIgnores NULLs
AVG(col)AverageIgnores NULLs (can mislead!)
Gotcha

AVG() ignores NULLs, which means it computes the average of existing values only. If NULLs represent zeros, use AVG(COALESCE(col, 0)).
