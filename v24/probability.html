🔬 Bayes' Theorem & Conditional Probability FOCUS

▼

Bayes' Theorem: P(A|B) = P(B|A) * P(A) / P(B)

In words: Posterior = (Likelihood x Prior) / Evidence

TermMeaning
Prior P(A)Belief before seeing data
Likelihood P(B|A)Probability of data under hypothesis
Evidence P(B)Overall probability of data
Posterior P(A|B)Updated belief after data

The Medical Test Problem: Disease affects 1%. Test is 99% accurate. You test positive. P(disease) = ?

P(+) = P(+|D)*P(D) + P(+|D')*P(D')
     = 0.99 * 0.01 + 0.01 * 0.99
     = 0.0198

P(D|+) = 0.0099 / 0.0198 = 50%
Why 50%, not 99%? In 10,000 people: 100 sick (99 test positive), 9,900 healthy (99 false positives). Of 198 positives, only 99 are truly sick. The base rate (1%) dominates.

Exercise: Bayes Calculation

⏱ 00:00

A spam filter has 95% sensitivity and 98% specificity. 20% of emails are spam. An email is flagged. What is P(spam)?

P(flag|spam) = 0.95, P(flag|not spam) = 0.02, P(spam) = 0.20 P(flag) = 0.95*0.20 + 0.02*0.80 = 0.19 + 0.016 = 0.206 P(spam|flag) = 0.19 / 0.206 = 92.2% Higher than the medical test because the base rate (20%) is much higher than 1%.

🎯 Expected Value & Variance FOCUS

▼

E[X] = sum of x * P(X=x) — the long-run average.

Linearity: E[aX + bY] = aE[X] + bE[Y] — works even when X, Y are dependent!

Variance: Var(X) = E[X²] - (E[X])²

// Die game: Pay $5. Roll 6 → $20, Roll 4-5 → $10, else → $0.
E[winnings] = (1/6)*20 + (2/6)*10 + (3/6)*0
            = 3.33 + 3.33 + 0 = $6.67
E[profit]   = 6.67 - 5 = $1.67  → Play!
Key Insight: Linearity of expectation is incredibly powerful. You can decompose complex expected values into simple pieces without worrying about dependencies between variables.

🧩 Classic Interview Puzzles FOCUS

Birthday Problem: 23 people for a 50%+ chance of a shared birthday. C(23,2) = 253 pairs, each with 1/365 chance.

Monty Hall: 3 doors, 1 car. You pick, host reveals a goat. Always switch — wins 2/3.

Coupon Collector: E[boxes for all n coupons] = n * H(n) where H(n) is the harmonic number. For n=50, about 225 boxes.

Approach: "At least one..." problems are almost always easier as 1 - P(nobody). Complement counting is your friend.

Bayes' Theorem

  • P(A|B) = P(B|A) * P(A) / P(B)
  • Base rate dominates with rare events
  • Medical test: 1% prevalence + 99% accuracy = only 50% PPV

Distributions

  • Binomial: n trials, count successes
  • Poisson: events per interval (lambda)
  • Normal: 68-95-99.7 rule
  • Exponential: memoryless, time between events

Expected Value

  • E[aX + bY] = aE[X] + bE[Y] (even dependent!)
  • Var(X+Y) = Var(X) + Var(Y) only if independent

Puzzles

  • Birthday: 23 people for 50%+
  • Monty Hall: always switch (2/3 win)
  • "At least one" = 1 - P(none)
