Machine Learning

3 conceptual challenges. Think through each one before revealing the answer. Each challenge has its own timer.

Challenge 1 — Choose the Right Model

Easy
Time:00:00

Think about it... You have a dataset with 50,000 rows and 200 features. Your target is binary (churn/no churn). The business needs an interpretable model they can explain to stakeholders. Which model would you choose and why? What if they instead wanted maximum predictive accuracy?

For interpretability: Logistic Regression. Coefficients directly show feature impact (log-odds), easy to explain to stakeholders. With 200 features, add L1 (Lasso) regularization to auto-select important features.

For max accuracy: Gradient Boosted Trees (XGBoost/LightGBM). They handle non-linear relationships, feature interactions, and mixed data types well. With 50K rows and 200 features, you have enough data to avoid overfitting with proper tuning.

Key tradeoff: You could use SHAP values on top of a GBM to get interpretability AND accuracy, giving the best of both worlds.

Challenge 2 — Calculate F1 Score

Medium
Time:00:00

Think about it... Your fraud detection model has these results on a test set: 90 True Negatives, 5 True Positives, 3 False Positives, 2 False Negatives. Calculate Precision, Recall, and F1 score. Is this model good enough for production? What metric matters most here?

Precision = TP / (TP + FP) = 5 / (5 + 3) = 0.625 (62.5%)
Recall = TP / (TP + FN) = 5 / (5 + 2) = 0.714 (71.4%)
F1 = 2 * (P * R) / (P + R) = 2 * (0.625 * 0.714) / (0.625 + 0.714) = 0.667 (66.7%)

Is it production-ready? Probably not. For fraud detection, Recall matters most — missing a fraud case (FN) is much more costly than flagging a legitimate transaction (FP). A recall of 71.4% means you're missing ~29% of fraud. You'd want to lower the classification threshold to catch more fraud, even at the cost of more false positives.

Challenge 3 — Spot the Data Leakage

Hard
Time:00:00

Think about it... A colleague builds a model to predict whether a patient will be readmitted within 30 days. They get 99% accuracy on the test set. Their features include: age, diagnosis code, number of medications, length of hospital stay, discharge disposition code, and "number of readmissions in past year." Where is the data leakage? Name all sources.

Leakage sources:

1. "Number of readmissions in past year" — This is the most obvious leak. If computed after the current visit, it directly encodes the target variable. Even if computed before, it may include information from the training period that wouldn't be available at prediction time.

2. "Discharge disposition code" — This is only known AFTER the patient is discharged, but you need to predict readmission at admission time (or during stay). This is a temporal leakage — using future information.

3. "Length of hospital stay" — Same problem: unknown at the time you'd want to make the prediction (at admission). Only known after discharge.

Fix: Only use features available at the time of prediction. If predicting at admission: use age, diagnosis, medications at admission, and historical data computed strictly before the current visit.
