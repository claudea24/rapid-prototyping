Challenge 1 — Deduplicate

Easy
Time:00:00

Use a set to remove duplicates from a list while preserving the original order. Input: [3, 1, 4, 1, 5, 9, 2, 6, 5, 3]

Hint: Use a set to track what you've seen, iterate through the list and only keep first occurrences. Or use dict.fromkeys() for a one-liner.
nums = [3, 1, 4, 1, 5, 9, 2, 6, 5, 3] # Method 1: Manual with set seen = set() result = [] for n in nums: if n not in seen: seen.add(n) result.append(n) # Method 2: One-liner (Python 3.7+ dict preserves order) result = list(dict.fromkeys(nums)) print(result) # [3, 1, 4, 5, 9, 2, 6]

Challenge 2 — GroupBy Revenue

Easy
Time:00:00

Use Pandas groupby + agg to calculate total revenue and average order size per region.

Hint: Use .groupby('region').agg() with a dictionary specifying different aggregation functions per column.
import pandas as pd df = pd.DataFrame({ 'region': ['East','West','East','West','East','North'], 'revenue': [100, 200, 150, 300, 80, 250], 'orders': [5, 8, 6, 12, 3, 10] }) result = df.groupby('region').agg( total_revenue=('revenue', 'sum'), avg_order_size=('revenue', 'mean'), total_orders=('orders', 'sum') ).reset_index() print(result) # region total_revenue avg_order_size total_orders # 0 East 330 110.000000 14 # 1 North 250 250.000000 10 # 2 West 500 250.000000 20

Challenge 3 — Matrix Multiply

Medium
Time:00:00

Use NumPy's @ operator to compute OLS regression coefficients. Given X (with intercept) and y, calculate (X'X)^{-1} X'y.

Hint: X.T is the transpose. Use np.linalg.inv() for the inverse. Chain them: inv(X.T @ X) @ X.T @ y
import numpy as np X = np.array([ [1, 2], [1, 3], [1, 5], [1, 7], [1, 8] ], dtype=float) y = np.array([3, 5, 8, 12, 14], dtype=float) # OLS formula: beta = (X'X)^-1 X'y beta = np.linalg.inv(X.T @ X) @ X.T @ y print(f"Intercept: {beta[0]:.2f}") # ~-0.10 print(f"Slope: {beta[1]:.2f}") # ~1.82 # Alternative (more numerically stable): beta2 = np.linalg.lstsq(X, y, rcond=None)[0]

Challenge 4 — Two Sum

Medium
Time:00:00

Given a list of integers and a target, return the indices of two numbers that add up to the target. Use a hash map for O(n) time.

Hint: As you iterate, store each number's index in a dict. For each number, check if (target - num) is already in the dict.
def two_sum(nums, target): """Return indices of two numbers that sum to target.""" seen = {} # value -> index for i, num in enumerate(nums): complement = target - num if complement in seen: return [seen[complement], i] seen[num] = i return [] # Time: O(n), Space: O(n) print(two_sum([2, 7, 11, 15], 9)) # [0, 1] print(two_sum([3, 2, 4], 6)) # [1, 2] print(two_sum([3, 3], 6)) # [0, 1]

Challenge 5 — Mutable Trap

Hard
Time:00:00

This function has a classic Python bug: mutable default argument. Each call to append_to accumulates items because the default list is shared. Fix it.

Hint: Use None as the default, then create a new list inside the function if target is None. Default mutable arguments are evaluated once at function definition time, not per call.
# FIXED: Use None sentinel pattern def append_to(element, target=None): if target is None: target = [] target.append(element) return target print(append_to(1)) # [1] print(append_to(2)) # [2] print(append_to(3)) # [3] # Why? Default arguments are evaluated ONCE at # function definition time. A mutable default (like []) # is shared across all calls. Using None + creating # a new list inside the function fixes this.
