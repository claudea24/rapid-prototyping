Machine Learning — Company Interview Patterns

From supervised learning to cross-validation — see which companies focus on which ML topics.

3.1 Supervised Learning Algorithms MetaGoogleAmazon

▾
All companies test supervised learning fundamentals. Expect "when would you use X vs Y?" questions.

Linear Regression fits a straight line by minimizing squared residuals. Logistic Regression models log-odds for classification. Decision Trees recursively partition feature space. Random Forest corrects high variance through bagging. Gradient Boosting builds trees sequentially on residual errors.

AlgorithmStrengthsWeaknessesWhen to Use
Linear RegressionInterpretable, fastAssumes linearityBaseline; interpretability paramount
Logistic RegressionProbabilistic, interpretableLinear boundary onlyBinary classification + explainability
Decision TreeNonlinear, no scalingHigh variance, overfitsQuick exploration
Random ForestLow variance, robustLess interpretableGeneral-purpose accuracy
Gradient BoostingState-of-the-art tabularOverfitting riskMax accuracy needed
SVMHigh-dim, kernel flexibilitySlow, opaqueText; small/medium datasets
KNNNo training, complex boundariesSlow inferenceSmall datasets; prototyping
Key Insight

Linear regression is not just a model — it is a diagnostic framework. You learn more from its residual plots than from its predictions.

3.2 Unsupervised Learning Google

▾
Google focuses on clustering at scale. Expect K-Means, DBSCAN, and PCA questions.

Unsupervised learning seeks structure without labels. K-Means creates spherical clusters. DBSCAN finds arbitrary-shaped clusters and identifies outliers. PCA finds axes of maximum variance for dimensionality reduction.

AlgorithmTypeKey PropertyUse When
K-MeansClusteringFast, spherical clustersKnown k, convex clusters
HierarchicalClusteringDendrogram explorationUnknown k, want hierarchy
DBSCANClusteringArbitrary shapes, outliersNo k needed, noise expected
PCADim. ReductionLinear, fastFeature reduction, preprocessing
t-SNE / UMAPDim. ReductionNonlinear, local structure2D visualization only
Key Insight

t-SNE and UMAP preserve local neighborhood structure but distort global distances. Use them to see your data, never to measure it.

3.4 Bias-Variance Tradeoff MetaGoogleAmazon

▾
All companies test this fundamental concept. Expect the dartboard analogy and regularization questions.

Total Error = Bias² + Variance + Irreducible Noise. High bias = underfitting (systematically wrong). High variance = overfitting (captures noise as signal).

ScenarioBiasVarianceSymptomFix
UnderfittingHighLowPoor on train AND testMore features, complex model
Sweet spotLowLowGood on bothKeep it
OverfittingLowHighGreat train, poor testRegularization, more data

L1 (Lasso) adds sum of |coefficients| → sparsity, feature selection. L2 (Ridge) adds sum of coefficients² → shrinkage, stability. Elastic Net combines both.

Key Insight

The gap between training and test error is a direct measure of overfitting. When that gap widens, simplify or regularize.

3.6 Cross-Validation & Data Leakage Google

▾
Google values methodological rigor. Expect questions about k-fold CV, time series CV, and leakage detection.

K-Fold CV: Split data into k folds. Each iteration: one fold as test, k-1 as train. Every point appears in both roles exactly once.

Time Series CV: Standard k-fold is fatal for time series — it trains on future data. Respect temporal order: training set only includes data before the test period.

from sklearn.model_selection import cross_val_score
scores = cross_val_score(model, X, y, cv=5, scoring='f1')
print(f"Mean F1: {scores.mean():.3f} (+/- {scores.std():.3f})")

Data Leakage: The most insidious form is fitting preprocessing on the full dataset before splitting. If you standardize using the mean/std of the entire dataset, the model implicitly "knows" something about the test data.

Key Insight

If performance seems "too good to be true," it almost certainly is. The most common culprit is data leakage. Audit every feature: "Would I have this information at the moment of prediction?"
