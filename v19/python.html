2.2 Pandas for Data Manipulation Meta

▾
Meta heavily tests Pandas in data manipulation interviews. Expect groupby, merge, and pivot questions.

A Pandas DataFrame is a two-dimensional, labeled data structure. The golden rule: if you are writing a for-loop over rows, you are almost certainly doing it wrong.

OperationMethodWhat It Does

import pandas as pd

summary = df.groupby('region')['revenue'].agg(
    total_revenue='sum',
    avg_order='mean',
    num_transactions='count'
)
Key Insight

If you're writing a for loop over DataFrame rows, use vectorized operations or groupby + agg instead.

2.3 NumPy for Numerical Computing Google

▾
Google tests numerical computing fundamentals. Expect vectorization and broadcasting questions.

NumPy's core contribution is the ndarray — a homogeneous, contiguous block of memory enabling vectorized operations. Speedups of 10–100x are typical over Python loops.

Broadcasting lets arrays of different shapes work together. Dimensions are compared right-to-left; they are compatible if they are equal or one is 1. * is element-wise product, while @ is the true matrix product.

import numpy as np

X = np.array([[1, 2, 5],
              [1, 3, 8],
              [1, 5, 12]], dtype=float)
y = np.array([10, 15, 24], dtype=float)

# Closed-form OLS: beta = (X^T X)^-1 X^T y
beta = np.linalg.inv(X.T @ X) @ X.T @ y
Key Insight

When you see for i in range(len(arr)) with NumPy arrays, stop and ask: "Can I express this as a vectorized operation?" Almost always, yes.

# Hash map approach: O(n)
def two_sum(nums, target):
    seen = {}  # value -> index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i

print(two_sum([2, 7, 11, 15], 9))  # [0, 1]
Key Insight

The hash map pattern is the single most important tool in coding interviews. Whenever brute-force uses a nested loop where the inner loop is searching, replace that search with a dictionary lookup.

2.5 Python Gotchas Google

▾
Google loves testing tricky Python knowledge. Expect questions about mutable defaults, copying, and is vs ==.
GotchaWhat HappensThe Fix
Mutable defaultsdef f(x=[]): list shared across all callsUse None sentinel
Shallow copyInner objects still sharedUse copy.deepcopy()
Floor division-7 // 2 = -4 (rounds toward -inf)Use int(a/b) for truncation
is vs ==is checks identity, == checks valueUse is only for None
# Mutable default argument bug
def append_to(item, target=[]):
    target.append(item)
    return target

print(append_to(1))   # [1]
print(append_to(2))   # [1, 2] -- SAME list!

# The fix: use None sentinel
def append_to(item, target=None):
    if target is None:
        target = []
    target.append(item)
    return target
Key Insight

Use is only to check for None (i.e., if x is None). For everything else, use ==. Relying on is for integers or strings is a bug waiting to happen.
