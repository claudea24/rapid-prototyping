Common Coding Patterns

Key

Patterns that appear repeatedly in coding interviews.

# Two pointers
def two_sum_sorted(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo < hi:
        s = nums[lo] + nums[hi]
        if s == target: return [lo, hi]
        elif s < target: lo += 1
        else: hi -= 1

# Hash map frequency count
from collections import Counter
freq = Counter(words)
top_k = freq.most_common(3)

# Sliding window
def max_sum_window(arr, k):
    w = sum(arr[:k])
    best = w
    for i in range(k, len(arr)):
        w += arr[i] - arr[i - k]
        best = max(best, w)
    return best

String & List Tricks

Tip

Pythonic one-liners that save time and impress interviewers.

# List comprehension with filter
evens = [x for x in nums if x % 2 == 0]

# zip to pair and iterate
for name, score in zip(names, scores):
    print(f"{name}: {score}")

# sorted with key
sorted(students, key=lambda s: s['gpa'], reverse=True)

# string join & split

'a,b,c'.split(',')          # ['a', 'b', 'c']

# enumerate for index + value
for i, val in enumerate(items):
    print(i, val)

Gotchas

Watch

Common Python pitfalls that trip up even experienced developers.

  • Mutable default arguments — def f(x=[]) shares the same list across calls. Use None and create inside the function.
  • Shallow vs deep copy — b = a[:] or copy() only copies the top level. Nested objects still share references. Use copy.deepcopy() for nested structures.
  • Integer division — // floors toward negative infinity: -7 // 2 == -4, not -3.
  • is vs == — is checks identity (same object in memory), == checks equality (same value). Use == for value comparison.
# Mutable default — BAD
def append_to(val, lst=[]):
    lst.append(val)
    return lst

# GOOD
def append_to(val, lst=None):
    if lst is None:
        lst = []
    lst.append(val)
    return lst
