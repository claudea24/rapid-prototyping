Structuring a Case

Tip
  1. Clarify the question & context
  2. Define the goal and success metric
  3. Break down into components
  4. Prioritize the highest-impact area
  5. Propose a solution with trade-offs
  6. Summarize and state next steps

Metric Change Questions

Key

"Metric X dropped 10%" — follow these steps:

  1. Clarify the metric definition & check tracking
  2. Segment by dimensions (platform, region, user type)
  3. Check internal changes (launches, bugs, experiments)
  4. Check external factors (seasonality, competitors)
  5. Quantify impact and recommend action

Market Sizing

Tip
  • Top-down: Start from total population, narrow down
  • Bottom-up: Build from unit economics upward
  • Assumptions: State clearly, keep reasonable
  • Round numbers: Simplify math, focus on logic

Experiment Design

Key
  • Randomization: Proper unit of assignment
  • Duration: Sufficient for statistical power
  • Guardrails: Monitor for regressions in key metrics
  • Network effects: Account for interference between groups

Communication Tips

Tip
  • Lead with your answer: State the recommendation first
  • Structure: Use a clear framework throughout
  • Quantify: Back up points with numbers
  • Trade-offs: Acknowledge pros and cons
