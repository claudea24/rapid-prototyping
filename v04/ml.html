Machine Learning

Algorithms, evaluation metrics, and model selection

Study cards

Supervised Algorithms

Key
  • Linear Regression
    Continuous target, assumes linearity
  • Logistic Regression
    Classification via sigmoid; outputs probabilities
  • Decision Trees
    Splits on feature thresholds; interpretable but prone to overfit
  • Random Forest
    Bagged ensemble of trees; reduces variance
  • Gradient Boosted Trees
    Sequential boosting; XGBoost, LightGBM
  • SVM
    Maximizes margin; kernel trick for non-linear boundaries
  • KNN
    Instance-based; classify by majority of k nearest neighbors

Unsupervised Algorithms

Key
  • K-Means
    Partition into k clusters by centroid distance; choose k via elbow method
  • Hierarchical Clustering
    Agglomerative (bottom-up) or divisive; produces dendrogram
  • DBSCAN
    Density-based; finds arbitrary shapes, handles noise as outliers
  • PCA
    Dimensionality reduction via orthogonal projections that maximize variance
  • t-SNE / UMAP
    Non-linear embedding for visualization; preserves local structure

Evaluation Metrics

Key
  • Accuracy
    Correct / Total — misleading on imbalanced data
  • Precision
    TP / (TP + FP) — "of predicted positives, how many are correct?"
  • Recall
    TP / (TP + FN) — "of actual positives, how many did we catch?"
  • F1 Score
    Harmonic mean of precision & recall
  • AUC-ROC
    Area under TPR vs FPR curve; threshold-independent
  • RMSE / MAE
    Regression error metrics; RMSE penalizes large errors more
F1 = 2 · (P · R) / (P + R)

Bias-Variance Tradeoff

Key

Bias — error from overly simple assumptions; model underfits.

Variance — error from sensitivity to training data; model overfits.

Total Error = Bias² + Variance + Noise

Regularization constrains model complexity:

  • L1 (Lasso)
    Adds |w| penalty; drives coefficients to zero (feature selection)
  • L2 (Ridge)
    Adds w² penalty; shrinks coefficients evenly

Feature Engineering

Tip
  • One-Hot Encoding
    Binary columns per category; use drop_first to avoid multicollinearity
  • Label Encoding
    Integer mapping; suitable for ordinal features or tree models
  • Scaling
    StandardScaler (z-score) or MinMaxScaler (0-1); required for distance-based models
  • Log Transform
    Reduces right skew; useful for revenue, counts
  • Feature Crosses
    Multiply/combine features to capture interactions
  • Target Encoding
    Replace category with mean of target; use with regularization to avoid leakage

Cross-Validation

Watch
  • k-Fold CV
    Split data into k folds; train on k-1, validate on 1; rotate
  • Stratified k-Fold
    Preserves class distribution in each fold; essential for imbalanced data
  • Time Series CV
    Expanding or sliding window; never use future data to predict past
  • Data Leakage
    Fitting preprocessors (scaler, encoder) on full data before splitting leaks test info into training
Always fit_transform(train) then transform(test)

Practice Questions

Random Forest primarily reduces which type of error?
Random Forest uses bagging (bootstrap aggregation) to average many decorrelated trees, which reduces variance without significantly increasing bias.
What is the risk of fitting a scaler on the full dataset before cross-validation?
Fitting the scaler on all data lets information from the validation fold leak into training, causing data leakage. Always fit on training data only, then transform the validation/test set.
