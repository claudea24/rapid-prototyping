# Common

JOIN Types

  • INNER JOIN — matching rows in both tables
  • LEFT JOIN — all rows from left + matches
  • RIGHT JOIN — all rows from right + matches
  • FULL OUTER JOIN — all rows from both
  • CROSS JOIN — cartesian product
  • SELF JOIN — table joined to itself
# Legendary

Window Functions

Compute values across a set of rows related to the current row, without collapsing them.

SELECT
  name,
  department,
  salary,
  RANK() OVER (
    PARTITION BY department
    ORDER BY salary DESC
  ) AS dept_rank
FROM employees;
# Rare

Aggregation Gotchas

  • COUNT(*) counts all rows; COUNT(col) skips NULLs
  • AVG ignores NULLs — denominator shrinks
  • Non-aggregated columns in SELECT must be in GROUP BY
  • HAVING filters groups; WHERE filters rows
  • DISTINCT inside aggregates: COUNT(DISTINCT col)
# Rare

CTEs & Subqueries

CTEs improve readability and can reference prior CTEs.

WITH monthly AS (
  SELECT
    user_id,
    DATE_TRUNC('month', created_at) AS mth,
    SUM(amount) AS total
  FROM orders
  GROUP BY 1, 2
)
SELECT
  mth,
  total,
  LAG(total) OVER (ORDER BY mth) AS prev
FROM monthly;
# Common

Performance Tips

  • Add indexes on columns used in WHERE and JOIN
  • Avoid SELECT * — fetch only needed columns
  • Use EXISTS instead of IN for large subqueries
  • Filter early with WHERE to reduce row counts
  • Check execution plans with EXPLAIN ANALYZE
