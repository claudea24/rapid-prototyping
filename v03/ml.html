The ML Forge

Step into the forge where raw data is smelted into predictive power. Train your algorithms on six floors of supervised learners, unsupervised explorers, and the legendary bias-variance tradeoff.

Concept Cards

Supervised Algorithms

Common
▼  Expand
  • Linear Regression — fits a line to minimize MSE. Assumes linearity, independence, homoscedasticity. Coefficients are interpretable.
  • Logistic Regression — sigmoid function maps linear output to [0, 1] probability. Decision boundary is linear. Uses log-loss.
  • Decision Trees — recursive splits on features using Gini impurity or information gain. Prone to overfitting without pruning.
  • Random Forest — ensemble of bagged trees with random feature subsets. Reduces variance via averaging. Robust to outliers.
  • XGBoost — gradient-boosted trees trained sequentially. Each tree corrects residuals of the previous. Regularized (L1/L2).
  • SVM — finds the maximum-margin hyperplane. Kernel trick enables non-linear boundaries. C controls margin softness.
  • KNN — classifies by majority vote of k nearest neighbors. Lazy learner (no training phase). Sensitive to feature scaling.

Unsupervised Algorithms

Common
▼  Expand
  • K-Means — iteratively assigns points to nearest centroid, then recomputes centroids. Choose k via elbow method or silhouette score.
  • Hierarchical Clustering — agglomerative (bottom-up) or divisive (top-down). Produces a dendrogram. No need to prespecify k.
  • DBSCAN — density-based clustering. Finds arbitrary-shaped clusters and identifies outliers as noise. Params: eps, min_samples.
  • PCA — projects data onto principal components (directions of max variance). Used for dimensionality reduction. Linear method.
  • t-SNE / UMAP — non-linear dimensionality reduction for visualization. t-SNE preserves local structure; UMAP is faster and preserves more global structure.

Evaluation Metrics

Rare
▼  Expand
  • Accuracy — correct predictions / total. Misleading on imbalanced datasets (e.g., 99% negative class).
  • Precision — TP / (TP + FP). How many predicted positives are actually positive. Prioritize when false positives are costly.
  • Recall — TP / (TP + FN). How many actual positives are found. Prioritize when false negatives are costly (e.g., cancer screening).
  • F1 Score — harmonic mean of precision and recall. Balances both when classes are imbalanced.
  • AUC-ROC — area under the ROC curve (TPR vs FPR). Threshold-independent measure of model discrimination. 0.5 = random, 1.0 = perfect.
  • RMSE / MAE — regression metrics. RMSE penalizes large errors more (squared). MAE is more robust to outliers. Both in original units.

Bias-Variance Tradeoff

Legendary
▼  Expand
  • Bias (Underfitting) — error from overly simplistic models that miss patterns. High bias = high training error. Example: linear model on non-linear data.
  • Variance (Overfitting) — error from models too sensitive to training noise. High variance = low train error, high test error. Example: deep unpruned tree.
  • The Tradeoff — Total Error = Bias² + Variance + Irreducible Noise. Increasing model complexity lowers bias but raises variance. Sweet spot minimizes total error.
  • L1 Regularization (Lasso) — adds |w| penalty. Drives coefficients to zero, performing feature selection. Produces sparse models.
  • L2 Regularization (Ridge) — adds w² penalty. Shrinks coefficients toward zero but never exactly zero. Handles multicollinearity well.
  • Elastic Net — combines L1 + L2. Balances sparsity and coefficient shrinkage. Useful when features are correlated.

Feature Engineering

Rare
▼  Expand
  • One-Hot Encoding — creates binary columns for each category. Use for nominal (unordered) features. Watch out for high-cardinality explosion.
  • Label Encoding — maps categories to integers. Suitable for ordinal features (e.g., low/medium/high). Tree models handle this well.
  • Scaling — StandardScaler (zero mean, unit variance) for Gaussian-like data. MinMaxScaler for bounded ranges. Required by SVM, KNN, and neural nets.
  • Log Transform — compresses right-skewed distributions. Useful for features like income, price, or counts. Apply log(x + 1) for zeros.
  • Feature Crosses — multiply or combine features to capture interactions. Example: city × device_type. Adds non-linearity to linear models.
  • Target Encoding — replaces categories with mean of the target variable. Powerful but prone to data leakage; use with cross-validation folds.

Cross-Validation

Rare
▼  Expand
  • k-Fold CV — splits data into k folds, trains on k-1, validates on the held-out fold. Repeats k times. Standard: k = 5 or 10.
  • Stratified k-Fold — preserves class distribution in each fold. Essential for imbalanced datasets. Default in scikit-learn for classification.
  • Time Series CV — expanding or sliding window splits that respect temporal order. Never use future data to predict the past.
  • Data Leakage Warning — fitting preprocessing (scaling, encoding, imputation) on the full dataset before splitting leaks test information into training. Always fit transformers inside the CV loop.
