Q1
What data structure would you use for O(1) lookup?
A) A sorted list with binary search — O(log n) is close enough.
B) A dictionary (hash map) — average O(1) lookup by key.
C) A set — it's fast for membership checks.
↪
Good. What happens when there are hash collisions? Does it stay O(1)?
❌ Mediocre Answer

"Collisions don't really happen in Python. Dictionaries are always O(1)."

✅ Great Answer

"Hash collisions are handled via open addressing in CPython. When two keys hash to the same slot, Python probes for the next open slot. Average case stays O(1), but worst case degrades to O(n) if many keys collide. Sets also use hash tables internally, so both dict and set give O(1) average lookup. The key requirement is that keys must be hashable (immutable)."

Q2
How does groupby work in Pandas? Walk me through it.
A) It splits the dataframe into groups, then you apply a function to each.
B) Split-Apply-Combine: split by key, apply aggregation, combine results back into a DataFrame.
C) It sorts the data by the group column, then iterates through it.
↪
What's the difference between .agg() vs .apply() after a groupby?
❌ Mediocre Answer

"They're pretty much interchangeable. agg is just shorter syntax."

✅ Great Answer

".agg() applies column-level aggregation functions (sum, mean, etc.) and can apply different functions to different columns — it's optimized and fast. .apply() passes the entire sub-DataFrame to a custom function, giving full flexibility but it's slower because it can't be vectorized. Rule of thumb: use .agg() for standard aggregations, .apply() only when you need custom logic that operates on multiple columns together."

Q3
Solve Two Sum: given an array of integers and a target, return indices of two numbers that add up to the target.
A) Brute force: check every pair with two nested loops. O(n²).
B) Hash map: for each number, check if (target - num) is already in the map. O(n).
C) Sort the array first, then use two pointers. O(n log n).
↪
What's the time and space complexity of your hash map approach?
❌ Mediocre Answer

"It's O(n) time. Space is... also O(n) I guess? Because of the hash map."

✅ Great Answer

"O(n) time — one pass through the array. O(n) space for the hash map in the worst case (no match until the last element). The tradeoff vs brute force: we spend O(n) extra memory to go from O(n²) to O(n) time. The two-pointer approach is O(n log n) time and O(1) space, but it loses original indices when sorting — so for returning indices, hash map is better."

Q4
What's wrong with def f(x=[]):? Why is this a common Python gotcha?
A) Nothing — it creates a new empty list each time the function is called.
B) The default list is created ONCE at definition time and shared across all calls — it mutates between calls.
C) It's a performance issue — creating lists as defaults is slow.
↪
Related: what's the difference between deep copy and shallow copy?
❌ Mediocre Answer

"Deep copy copies everything, shallow copy just copies the top level. Use deep copy to be safe."

✅ Great Answer

"A shallow copy (copy.copy() or list.copy()) creates a new outer container but the nested objects still reference the originals. A deep copy (copy.deepcopy()) recursively copies everything. For the mutable default argument fix, use def f(x=None): x = x if x is not None else [] — this creates a fresh list each call. The core principle: in Python, default arguments are evaluated once at function definition, not at each call."
