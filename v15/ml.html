F1

Supervised Learning

+75 XP ▼

Linear Regression fits a hyperplane minimizing squared residuals. Assumes linearity, normal residuals, and no multicollinearity. It's a diagnostic framework — learn more from residual plots than predictions.

Logistic Regression models log-odds as a linear function. The sigmoid squashes output to (0,1). Coefficients are interpretable as change in log-odds per unit change.

Decision Trees recursively partition feature space using Gini impurity or entropy. Interpretable but high variance — small data changes can produce different trees.

Random Forest corrects variance through bagging — many trees on random subsets, averaged. The "wisdom of crowds."

Gradient Boosting builds trees sequentially, each correcting prior errors via gradient descent in function space. XGBoost, LightGBM, CatBoost dominate tabular data.

SVM maximizes the margin between classes. The kernel trick maps to higher dimensions for nonlinear problems. Elegant but scales poorly.

KNN is a lazy learner — stores training data, finds k nearest neighbors at prediction time. No training phase but slow inference.

F2

Unsupervised Learning

+75 XP ▼

K-Means: Alternates assigning points to nearest centroid and recomputing centroids. Use elbow method or silhouette score to choose k.

Hierarchical Clustering: Builds a dendrogram. Cut at desired level for clusters. Explore structure at multiple granularities.

DBSCAN: Defines clusters as dense regions. Core points within ε form clusters; everything else is noise. No k required.

PCA: Finds axes of maximum variance (eigenvectors of covariance matrix). Linear, fast, deterministic dimensionality reduction.

t-SNE / UMAP: Nonlinear dimensionality reduction for visualization. Preserve local structure but distort global distances.

Evaluation Metrics

+75 XP ▼

Accuracy is misleading with imbalanced data — a fraud detector predicting "not fraud" always achieves 99.5% accuracy while catching zero fraud.

MetricIdeaUse When
Accuracy(TP+TN) / totalBalanced datasets

RecallTP / (TP+FN)FN is costly (disease detection)
F1 ScoreHarmonic mean of P & RBalance precision and recall
AUC-ROCArea under TPR vs FPRComparing models, threshold-independent
RMSE√(mean squared errors)Penalize large errors (regression)

Key Insight
Before choosing a metric, always ask: "What is the cost of a false positive versus a false negative?" The answer drives your metric, threshold, model, and stakeholder presentation.
F4

Bias-Variance Tradeoff

+100 XP ▼

The Dartboard Analogy: High bias = darts cluster tight but away from bullseye (underfitting). High variance = darts scatter widely around bullseye (overfitting). Sweet spot = tight cluster on bullseye.

Total Error = Bias² + Variance + Irreducible Noise

ScenarioBiasVarianceSymptomFix
UnderfittingHighLowPoor on train AND testMore features, complex model
Sweet spotLowLowGood on bothKeep it
OverfittingLowHighGreat train, poor testRegularize, more data

L1 (Lasso): Adds |coefficients| → sparsity, drives coefficients to zero (feature selection).
L2 (Ridge): Adds coefficients² → shrinks all toward zero, stability.
Elastic Net: Combines both.

F6

Cross-Validation & Data Leakage

+100 XP ▼

K-Fold CV: Split data into k folds. Each iteration: one fold as test, k−1 as train. Every point appears in both roles exactly once.

from sklearn.model_selection import cross_val_score
scores = cross_val_score(model, X, y, cv=5, scoring='f1')
print(f"Mean F1: {scores.mean():.3f} (+/- {scores.std():.3f})")

Time Series CV: Standard k-fold is fatal for time series — it trains on future data. Use expanding or sliding window: training set only includes data before the test period.

Data Leakage: Fitting preprocessing on the full dataset before splitting. If you standardize using the full dataset's mean/std (including test), the model implicitly "knows" test data.

Rule: Everything that learns from data must be fit inside the training fold only. Use Pipeline.

Example: Hospital readmission model — computed features using current visit data and scaled on full dataset. Reported AUC: 94%. Production AUC: 68%. Two leaks, 26-point drop.

Key Insight
If performance seems "too good to be true," it almost certainly is. The most common culprit is data leakage. Audit every feature: "Would I have this information at the moment of prediction?"
