1. (Chapter I — SQL)

What does COUNT(*) count in a SQL query?

  • (a) Only non-NULL values in the first column
  • (b) All rows, including those with NULL values
  • (c) Only distinct rows
  • (d) All columns in each row
Answer

COUNT(*) counts every row in the result set regardless of NULL values. In contrast, COUNT(column) only counts non-NULL values in that specific column.

2. (Chapter I — SQL)

Which clause filters rows before grouping in a SQL query?

  • (a) WHERE
  • (b) HAVING
  • (c) GROUP BY
  • (d) ORDER BY
Answer

WHERE filters individual rows before any aggregation occurs. HAVING filters after grouping, operating on the results of aggregate functions.

3. (Chapter I — SQL)

What is the key difference between RANK() and DENSE_RANK()?

  • (a) RANK() is faster to compute
  • (b) DENSE_RANK() ignores NULL values
  • (c) RANK() leaves gaps after ties; DENSE_RANK() does not
  • (d) They are identical in behavior
Answer

When two rows tie for rank 2, RANK() assigns both rank 2 and skips to rank 4, leaving a gap. DENSE_RANK() assigns both rank 2 and continues to rank 3 with no gap.

4. (Chapter I — SQL)

Can you use a column alias defined in SELECT within the WHERE clause of the same query?

  • (a) Yes, aliases are available everywhere in the query
  • (b) Yes, but only in MySQL
  • (c) Yes, but only if the alias is numeric
  • (d) No, WHERE is evaluated before SELECT
Answer

In standard SQL, the logical order of evaluation processes WHERE before SELECT, so aliases defined in the SELECT clause are not yet available. You must repeat the expression or use a subquery.

5. (Chapter I — SQL)

When is EXISTS generally preferred over IN for subqueries?

  • (a) When the subquery returns exactly one row
  • (b) When the subquery is large, because EXISTS short-circuits on the first match
  • (c) When you need to check for NULL values in the subquery
  • (d) IN is always preferred over EXISTS
Answer

EXISTS returns TRUE as soon as it finds the first matching row and stops scanning, making it more efficient for large subqueries. IN may evaluate the entire subquery result set before filtering.

6. (Chapter II — Python)

What is the average time complexity of looking up a key in a Python dictionary?

  • (a) O(1)
  • (b) O(log n)
  • (c) O(n)
  • (d) O(n log n)
Answer

Python dictionaries are implemented as hash tables, providing average-case O(1) lookup. In the worst case (many hash collisions), it degrades to O(n), but this is rare in practice.

7. (Chapter II — Python)

Which method is most idiomatic for transforming every element of a list in Python?

  • (a) A for loop appending to a new list
  • (b) Using map() with a lambda
  • (c) A list comprehension
  • (d) Using reduce()
Answer

List comprehensions are considered the most Pythonic approach for element-wise transformations. They are concise, readable, and generally faster than map() with a lambda or manual loops.

8. (Chapter II — Python)

What is the danger of using a mutable default argument (e.g., def f(x=[])) in Python?

  • (a) It raises a SyntaxError
  • (b) The default object is shared across all calls, so mutations persist between invocations
  • (c) It creates a new list on every call, wasting memory
  • (d) It forces the function to return None
Answer

Default arguments are evaluated once at function definition time, not at each call. A mutable default like a list is shared across invocations, so appending to it in one call affects subsequent calls. The standard fix is to use None as the default and create the list inside the function.

9. (Chapter II — Python)

What does -1 // 2 evaluate to in Python 3?

  • (a) 0
  • (b) -0.5
  • (c) 0.5
  • (d) -1
Answer

The // operator performs floor division, rounding toward negative infinity. -1 / 2 equals -0.5, and the floor of -0.5 is -1. This often surprises programmers coming from languages where integer division truncates toward zero.

10. (Chapter III — Machine Learning)

Compared to a single decision tree, a Random Forest typically has:

  • (a) Higher variance and higher bias
  • (b) Lower variance and similar or slightly higher bias
  • (c) Higher variance and lower bias
  • (d) Identical bias and variance
Answer

By averaging many de-correlated trees (via bagging and feature subsampling), a Random Forest substantially reduces variance compared to a single tree while keeping bias roughly the same or slightly increasing it.

11. (Chapter III — Machine Learning)

Why is AUC-ROC often preferred over accuracy for imbalanced classification problems?

  • (a) AUC-ROC is faster to compute
  • (b) AUC-ROC penalises false positives more heavily
  • (c) AUC-ROC evaluates performance across all thresholds and is insensitive to class imbalance
  • (d) AUC-ROC only measures precision
Answer

Accuracy can be misleading when classes are imbalanced—a model predicting only the majority class can achieve high accuracy. AUC-ROC measures the trade-off between true positive rate and false positive rate across all classification thresholds, giving a more balanced view of model performance.

12. (Chapter III — Machine Learning)

Why does L1 (Lasso) regularisation tend to produce sparse models?

  • (a) The diamond-shaped constraint region has corners on the axes, pushing coefficients to exactly zero
  • (b) It squares the coefficients, magnifying small values
  • (c) It randomly drops features during training
  • (d) It uses a logarithmic penalty that favours zeros
Answer

L1 regularisation adds the sum of absolute values of coefficients to the loss function. Geometrically, the L1 constraint region is diamond-shaped with vertices on the coordinate axes, making it likely that the optimal solution lies at a corner where one or more coefficients are exactly zero.

13. (Chapter III — Machine Learning)

What problem arises from fitting a scaler (e.g., StandardScaler) on the entire dataset before splitting into train and test sets?

  • (a) The scaler becomes too slow
  • (b) Features become non-normally distributed
  • (c) The training set becomes too small
  • (d) Data leakage: test set statistics influence the training process
Answer

Fitting the scaler on the full dataset means the mean and standard deviation incorporate test set information. This causes data leakage, giving an overly optimistic estimate of model performance. The correct approach is to fit the scaler only on the training set and then transform both train and test sets.

14. (Chapter IV — Statistics)

What does a p-value represent in hypothesis testing?

  • (a) The probability that the null hypothesis is true
  • (b) The probability of observing data at least as extreme as the sample, assuming the null hypothesis is true
  • (c) The probability that the alternative hypothesis is true
  • (d) The probability of making a Type I error
Answer

A p-value is the probability of obtaining results at least as extreme as the observed data, given that the null hypothesis is true. It is not the probability that the null hypothesis itself is true or false.

15. (Chapter IV — Statistics)

A Type II error occurs when:

  • (a) We reject the null hypothesis when it is true
  • (b) We reject the alternative hypothesis when it is true
  • (c) We fail to reject the null hypothesis when it is false
  • (d) We accept the null hypothesis when it is true
Answer

A Type II error (false negative) occurs when a real effect exists but the test fails to detect it. The probability of a Type II error is denoted β, and statistical power is 1 − β.

16. (Chapter IV — Statistics)

The Central Limit Theorem states that:

  • (a) The sampling distribution of the sample mean approaches a normal distribution as sample size increases, regardless of the population distribution
  • (b) All populations are normally distributed
  • (c) Larger samples always have smaller means
  • (d) The sample mean equals the population mean for any sample size
Answer

The CLT is one of the most important results in statistics. It tells us that for sufficiently large sample sizes, the distribution of the sample mean is approximately normal—even if the underlying population is not—provided the population has finite variance.

17. (Chapter IV — Statistics)

In A/B testing, "peeking" at results before the sample size is reached is problematic because:

  • (a) It slows down the data collection process
  • (b) It changes the underlying distribution of the data
  • (c) It makes the test less powerful
  • (d) It inflates the false positive rate by performing multiple implicit comparisons
Answer

Each peek is effectively an additional hypothesis test. Without correction for multiple comparisons, repeatedly checking results dramatically inflates the Type I error rate—a test designed for α = 0.05 can have an effective false positive rate far exceeding 5%. Sequential testing methods or group sequential designs can mitigate this.

18. (Chapter V — Probability)

In the Monty Hall problem, after the host reveals a goat behind one of the other doors, what is the probability of winning the car if you switch?

  • (a) 1/3
  • (b) 2/3
  • (c) 1/2
  • (d) 1/4
Answer

When you initially pick a door, you have a 1/3 chance of being correct. The remaining 2/3 probability is concentrated on the other doors. Since the host always reveals a goat, switching gives you the full 2/3 probability. This is one of the most counter-intuitive results in probability.

19. (Chapter V — Probability)

A medical test has a sensitivity of 99% and a false positive rate of 5%. If the disease prevalence is 1%, what is the approximate probability that a person who tests positive actually has the disease?

  • (a) 99%
  • (b) 95%
  • (c) About 17%
  • (d) About 50%
Answer

By Bayes' theorem: P(Disease | Positive) = (0.99 × 0.01) / (0.99 × 0.01 + 0.05 × 0.99) = 0.0099 / 0.0594 ≈ 16.7%. The low prevalence means most positive results are false positives, illustrating the base rate fallacy.

20. (Chapter V — Probability)

If events A and B are independent, then P(A ∩ B) equals:

  • (a) P(A) × P(B)
  • (b) P(A) + P(B)
  • (c) P(A) + P(B) − P(A) × P(B)
  • (d) P(A|B) × P(A)
Answer

Independence means that the occurrence of one event does not affect the probability of the other. Mathematically, this is defined as P(A ∩ B) = P(A) × P(B). Option (c) describes P(A ∪ B), the union, not the intersection.

21. (Chapter V — Probability)

In the birthday problem, approximately how many people are needed in a room for a >50% chance that two share a birthday?

  • (a) 183
  • (b) 50
  • (c) 23
  • (d) 365
Answer

With just 23 people, the probability that at least two share a birthday exceeds 50%. This famously counter-intuitive result arises because the number of pairs grows quadratically: 23 people yield 23 × 22 / 2 = 253 possible pairs to check.

22. (Chapter VI — Case Studies)

When defining a "North Star metric" for a product, the most important characteristic is that it:

  • (a) Is easy to compute in real time
  • (b) Captures the core value the product delivers to users
  • (c) Correlates with revenue in the short term
  • (d) Can be A/B tested directly
Answer

A North Star metric should reflect the fundamental value your product provides. It aligns the entire organisation around long-term user value rather than short-term business metrics, serving as a compass for prioritisation and strategy.

23. (Chapter VI — Case Studies)

DAU (Daily Active Users) drops 10% overnight. What should be your first step?

  • (a) Check for data logging issues or pipeline failures before assuming a real product change
  • (b) Immediately roll back the most recent product change
  • (c) Run an A/B test to diagnose the cause
  • (d) Alert the marketing team to increase spend
Answer

Before investigating product causes, always verify the data is trustworthy. Logging bugs, pipeline failures, or tracking changes are common causes of apparent metric drops. Only after confirming data integrity should you segment users and investigate product or external factors.

24. (Chapter VI — Case Studies)

What is the purpose of a "guardrail metric" in an A/B test?

  • (a) To serve as the primary decision metric
  • (b) To measure the test's statistical power
  • (c) To track revenue during the experiment
  • (d) To ensure the experiment does not cause unacceptable harm to key metrics
Answer

Guardrail metrics are secondary metrics that must not degrade beyond a defined threshold. For example, while testing a change to improve engagement, you might use page load time or crash rate as guardrails to ensure the change does not harm user experience.

25. (Chapter VI — Case Studies)

Revenue per user is increasing while the number of rides (or transactions) is decreasing. The most likely explanation is:

  • (a) A data logging error in the rides pipeline
  • (b) Users are becoming less engaged overall
  • (c) Users are shifting toward fewer but higher-value transactions (e.g., longer rides, premium tiers)
  • (d) The pricing model has a bug that undercharges
Answer

When revenue per user rises while transaction volume falls, the mix is shifting toward higher-value transactions. This could be caused by price increases, premium tier adoption, longer rides, or a change in user demographics. Investigating the revenue distribution per transaction confirms this pattern.

26. (Chapter VI — Case Studies)

When designing a product metric for a two-sided marketplace, which consideration is most critical?

  • (a) Optimising solely for the supply side
  • (b) Balancing metrics for both supply and demand sides to avoid harming one at the expense of the other
  • (c) Focusing only on the metric that is easiest to measure
  • (d) Using a single metric that maximises short-term revenue
Answer

Two-sided marketplaces must maintain health on both sides. Optimising aggressively for one side (e.g., rider discounts) can degrade the experience for the other side (e.g., driver earnings), leading to attrition and a collapse of the marketplace equilibrium.
