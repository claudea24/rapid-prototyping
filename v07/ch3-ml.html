Section 3.1

Supervised Learning Algorithms

Supervised learning is the most widely used family of machine learning techniques. The premise is simple: you have labeled data—inputs paired with known outputs—and you want to learn a mapping from inputs to outputs that generalizes to unseen examples. What makes this domain rich is the variety of ways we can learn that mapping, each with its own geometric intuition, assumptions, and failure modes.

Linear Regression

Linear regression fits a straight line (or hyperplane) through your data by minimizing the sum of squared residuals—the vertical distances between observed and predicted values. The model assumes a linear relationship between features and the target, that residuals are normally distributed and homoscedastic (constant variance), and that features are not highly collinear.

When does it fail? When the true relationship is nonlinear, when outliers dominate the squared-error objective, or when multicollinearity inflates coefficient variance. In interviews, emphasize that linear regression is not just a model—it is a diagnostic framework. You learn more from its residual plots than from its predictions.

Logistic Regression

Despite the name, logistic regression is a classification algorithm. It models the log-odds of belonging to the positive class as a linear function of features:

log(p / (1 − p)) = β0 + β1x1 + … + βnxn

The sigmoid function squashes this output to the (0, 1) range, yielding a probability. The decision boundary is linear in feature space, meaning logistic regression works well when classes are roughly separable by a line. Its coefficients are directly interpretable as the change in log-odds per unit change in a feature—a property beloved by stakeholders who want to understand why the model makes its predictions.

Decision Trees

A decision tree recursively partitions the feature space by choosing, at each node, the feature and threshold that best separates the target classes. “Best” is measured by impurity criteria: Gini impurity quantifies the probability of misclassifying a randomly chosen sample, while entropy measures the information content of the split. In practice they produce very similar trees, but Gini is slightly faster to compute (no logarithm).

Trees are wonderfully interpretable and handle nonlinear relationships, missing values, and mixed feature types naturally. Their downfall is variance—a small change in the data can produce a completely different tree structure. Left unchecked, they will memorize the training data.

Random Forest

A random forest corrects the high variance of individual trees through bagging (bootstrap aggregating). It trains many trees on random subsets of both rows and features, then averages their predictions (regression) or takes a majority vote (classification). The magic is that by decorrelating the trees—each sees a different slice of the data—the ensemble’s variance drops dramatically while bias stays roughly the same. This is the statistical equivalent of “the wisdom of crowds.”

Gradient Boosting

Where random forests build trees in parallel, gradient boosting builds them sequentially. Each new tree is trained on the residual errors of the previous ensemble—it learns to correct the mistakes its predecessors made. The “gradient” comes from the fact that we are performing gradient descent in function space, stepping toward lower loss one tree at a time. Implementations like XGBoost, LightGBM, and CatBoost add regularization, histogram-based splits, and GPU support, making gradient boosting the dominant algorithm in tabular-data competitions.

Support Vector Machines (SVM)

SVMs find the hyperplane that maximizes the margin—the distance to the nearest data points of each class (the “support vectors”). The intuition is geometric: a wider margin means greater confidence in the boundary. For nonlinear problems, the kernel trick implicitly maps features into a higher-dimensional space where a linear separator exists, without ever computing the transformation explicitly. SVMs are elegant but scale poorly to very large datasets, and their performance is sensitive to hyperparameters (C, gamma).

K-Nearest Neighbors (KNN)

KNN is the ultimate lazy learner—it stores the entire training set and defers all computation to prediction time. To classify a new point, it finds the k closest training points (by Euclidean distance or another metric) and takes a majority vote. There is no training phase, no learned parameters. This simplicity is its strength and its weakness: it captures complex boundaries effortlessly, but is slow at inference, sensitive to the curse of dimensionality, and requires careful feature scaling since distances are meaningless when features have wildly different ranges.

Algorithm Comparison

Algorithm Strengths Weaknesses When to Use
Linear Regression Interpretable, fast, coefficients reveal feature importance Assumes linearity, sensitive to outliers Baseline for regression; when interpretability is paramount
Logistic Regression Probabilistic output, interpretable coefficients, regularizable Linear decision boundary only Binary classification with explainability needs; medical/legal contexts
Decision Tree Handles nonlinearity, no scaling needed, highly interpretable High variance, prone to overfitting Quick exploration; when a visual decision path is needed
Random Forest Low variance, robust, handles missing values well Less interpretable, slower than single tree General-purpose; when you need accuracy without much tuning
Gradient Boosting State-of-the-art on tabular data, flexible loss functions Prone to overfitting if not regularized, slower to train Competitions and production where accuracy is king
SVM Effective in high-dimensional spaces, kernel flexibility Slow on large data, opaque, sensitive to hyperparameters Text classification; small-to-medium datasets with many features
KNN No training needed, captures complex boundaries Slow inference, curse of dimensionality, needs scaling Small datasets; anomaly detection; recommendation prototyping

Section 3.2

Unsupervised Learning

Unsupervised learning operates without labels. Instead of predicting a target, we seek to discover structure hidden in the data—clusters of similar observations, lower-dimensional representations, or latent patterns. These techniques are invaluable when labels are expensive or nonexistent.

K-Means Clustering

K-Means begins with k randomly placed centroids. It alternates two steps: (1) assign each point to its nearest centroid, and (2) recompute each centroid as the mean of its assigned points. This process repeats until assignments stabilize. The algorithm converges to a local optimum—different initializations can yield different results, which is why k-means++ uses smart seeding to improve outcomes.

The critical question is how to choose k. The elbow method plots the within-cluster sum of squares (inertia) against increasing values of k. You look for the “elbow”—the point where adding more clusters yields diminishing returns. The silhouette score offers a complementary view, measuring how similar each point is to its own cluster versus neighboring clusters.

Hierarchical Clustering

Instead of specifying k upfront, hierarchical clustering builds a tree of merges (agglomerative, bottom-up) or splits (divisive, top-down). The result is a dendrogram—a tree diagram where the height of each merge indicates the distance between the clusters being joined. You “cut” the dendrogram at the desired level to obtain a specific number of clusters. This approach is especially useful when you want to explore cluster structure at multiple granularities.

DBSCAN

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) is fundamentally different from K-Means. It defines clusters as dense regions separated by areas of low density. A point is a core point if at least minPts neighbors fall within a radius ε. Core points that are within ε of each other are linked into the same cluster, and non-core points near a core point are absorbed. Points that belong to no cluster are labeled as noise.

The beauty of DBSCAN is that it does not require specifying the number of clusters, it can discover clusters of arbitrary shape, and it naturally identifies outliers. The trade-off: choosing ε and minPts requires domain knowledge or a k-distance plot.

PCA (Principal Component Analysis)

PCA is a dimensionality reduction technique that finds new axes (principal components) along which the data exhibits maximum variance. Mathematically, it computes the eigenvectors of the covariance matrix; the eigenvalues tell you how much variance each component captures. By keeping only the top components, you project high-dimensional data into a lower space while preserving as much information as possible. PCA is linear—it cannot capture nonlinear manifolds—but it is fast, deterministic, and an essential preprocessing step for downstream models.

t-SNE and UMAP

t-SNE (t-distributed Stochastic Neighbor Embedding) and UMAP (Uniform Manifold Approximation and Projection) are nonlinear dimensionality reduction techniques designed primarily for visualization. They excel at preserving local neighborhood structure, making clusters visually apparent in 2D. However, both distort global distances—the relative positions and sizes of clusters in the plot do not reflect true relationships. Use them to see your data, never to measure it.

Example 3.1 — Customer Segmentation

An e-commerce company collects features on each customer: average order value, purchase frequency, days since last purchase, product category preferences, and session duration. After standardizing all features (essential because K-Means uses Euclidean distance), we run K-Means with k = 2 through 10 and examine the elbow plot.

The elbow appears at k = 4. Inspecting each cluster’s centroid reveals four distinct personas: Loyal High-Spenders (frequent, high AOV), Bargain Hunters (frequent but low AOV, spike during sales), Window Shoppers (high session duration but low conversion), and Churning Customers (long recency, declining frequency). The marketing team now tailors campaigns to each segment—loyalty rewards for the first group, flash-sale alerts for the second, conversion-optimization nudges for the third, and re-engagement emails for the fourth.

We validate with silhouette scores (0.42—reasonable for real-world data) and overlay a UMAP projection to confirm that the four groups form visually distinct regions.

Section 3.3

Evaluation Metrics

A model is only as good as the metric you use to judge it. Choosing the wrong metric is one of the most common—and most consequential—mistakes in applied machine learning. This section builds your intuition for why certain metrics exist and when each is appropriate.

Accuracy (and Why It Fails)

Accuracy is the proportion of correct predictions. It is intuitive and satisfying, but dangerously misleading with imbalanced data. Consider a fraud detection dataset where 99.5% of transactions are legitimate. A model that predicts “not fraud” for every transaction achieves 99.5% accuracy while catching zero fraud. Accuracy tells you how often the model is right; it says nothing about which kinds of errors it makes.

Precision and Recall

Precision answers: “Of all the items the model flagged as positive, how many actually were?” Recall answers: “Of all the actual positives, how many did the model catch?” These two metrics are in tension—improving one typically hurts the other.

Consider a spam filter. High precision, low recall means the filter rarely marks a legitimate email as spam (few false positives), but it lets many spam messages through (many false negatives). High recall, low precision means it catches nearly all spam, but frequently blocks legitimate messages. Which is worse? For most users, missing an important email from a client (false positive) is far costlier than seeing an occasional spam message. So you would optimize for precision. But for a security-critical system filtering phishing attacks, you would optimize for recall—you cannot afford to let threats through.

F1 Score

The F1 score is the harmonic mean of precision and recall:

F1 = 2 × (Precision × Recall) / (Precision + Recall)

Why the harmonic mean rather than the arithmetic mean? Because the harmonic mean punishes extreme imbalances. If precision is 1.0 and recall is 0.01, the arithmetic mean is a deceptively respectable 0.505, but the harmonic mean (F1) is only 0.0198—reflecting the reality that the model is essentially useless for finding positives.

AUC-ROC

The ROC curve plots the True Positive Rate (recall) against the False Positive Rate at every possible classification threshold. The AUC (Area Under the Curve) summarizes this curve as a single number between 0 and 1. An AUC of 0.5 means the model is no better than random guessing; 1.0 means perfect separation. AUC is threshold-independent—it tells you how well the model ranks positives above negatives, regardless of where you set the decision boundary. This makes it especially useful when you have not yet decided on a threshold or when comparing models.

Regression Metrics: RMSE vs. MAE

RMSE (Root Mean Squared Error) penalizes large errors disproportionately because it squares residuals before averaging. MAE (Mean Absolute Error) treats all errors linearly. If your application is particularly intolerant of big misses—say, predicting hospital wait times where a 3-hour error is much worse than three 1-hour errors—use RMSE. If all errors are equally bad per unit, MAE gives a more robust and interpretable measure.

Key Insight

Before choosing a metric, always ask: “What is the cost of a false positive versus a false negative?” The answer should drive every downstream decision—your metric, your threshold, your model, and how you present results to stakeholders. A model is not “good” or “bad” in the abstract; it is good or bad relative to the costs of its errors.

Section 3.4

The Bias-Variance Tradeoff

Every prediction error can be decomposed into three components: bias, variance, and irreducible noise. Understanding this decomposition is perhaps the single most important conceptual tool in machine learning.

The Dartboard Analogy

Imagine you are throwing darts at a target, and the bullseye represents the true answer.

  • High bias, low variance: Your darts cluster tightly together, but consistently away from the bullseye. The model has learned a systematically wrong pattern—it underfits.
  • Low bias, high variance: Your darts are centered around the bullseye on average, but scattered widely. The model is capturing noise as if it were signal—it overfits.
  • Low bias, low variance: Darts are tightly clustered around the bullseye. This is the sweet spot—the model generalizes well.

The Error Decomposition

Total Error = Bias² + Variance + Irreducible Noise

Bias measures how far the average prediction is from the truth. Variance measures how much predictions fluctuate across different training sets. Irreducible noise is the error inherent in the data itself—no model can eliminate it. The tradeoff arises because reducing bias (using a more complex model) typically increases variance, and vice versa.

Regularization: L1 and L2

Regularization explicitly constrains model complexity to manage this tradeoff.

L1 regularization (Lasso) adds the sum of absolute coefficient values to the loss function. Its key property is sparsity—it drives some coefficients exactly to zero, effectively performing feature selection. Use Lasso when you suspect many features are irrelevant.

L2 regularization (Ridge) adds the sum of squared coefficient values. It shrinks all coefficients toward zero but rarely eliminates any. The result is stability—it is especially effective when features are correlated.

Elastic Net combines both, blending L1’s sparsity with L2’s stability.

Example 3.2 — Complexity vs. Generalization

Suppose we are predicting house prices with polynomial regression and vary the degree from 1 to 15:

  • Degree 1 (linear): The model draws a straight line through curved data. Training error is high, test error is high. This is underfitting—high bias, low variance.
  • Degree 3–4 (sweet spot): The curve captures the true nonlinear trend without chasing noise. Training error is moderate, test error is at its minimum. Bias and variance are balanced.
  • Degree 15: The polynomial wiggles wildly to pass through every training point. Training error is near zero, but test error explodes. This is overfitting—low bias, high variance.

The gap between training and test error is a direct measure of overfitting. When you see that gap widening, it is time to simplify the model, add regularization, or gather more data.

Section 3.5

Feature Engineering

Feature engineering is the art of transforming raw data into representations that make machine learning algorithms work better. It is often the difference between a mediocre model and a great one—and frequently matters more than the choice of algorithm itself.

One-Hot Encoding

Nominal categorical variables (color, city, product category) have no inherent order. One-hot encoding creates a binary column for each category. This prevents the model from assuming that “red = 1, blue = 2, green = 3” implies blue is somehow “between” red and green.

For ordinal variables (education level, satisfaction rating), one-hot encoding throws away the ordering information. Instead, use integer encoding or map to a meaningful numeric scale that preserves the rank. 

# One-hot encoding with pandas
df = pd.get_dummies(df, columns=['city', 'color'], drop_first=True)

# Ordinal encoding — preserves rank
edu_order = {'high_school': 1, 'bachelors': 2, 'masters': 3, 'phd': 4}
df['education'] = df['education'].map(edu_order)

Feature Scaling

Scaling matters for any algorithm that relies on distances or gradients: KNN, SVM, logistic regression with gradient descent, PCA, and neural networks. Tree-based models (decision trees, random forests, gradient boosting) are immune because they split on thresholds, not magnitudes.

Standardization (z-score) centers features to mean zero and unit variance. Min-max scaling compresses features to [0, 1]. Use standardization as the default; min-max when you need bounded outputs or when the algorithm assumes inputs in a fixed range. 

from sklearn.preprocessing import StandardScaler, MinMaxScaler

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)     # fit on train only
X_test_scaled  = scaler.transform(X_test)           # transform test with train params

Log Transforms

Many real-world distributions are right-skewed: income, house prices, page views, time between events. Applying log(x + 1) compresses the long tail and makes the distribution more symmetric. This helps linear models (which assume normally distributed residuals) and reduces the outsized influence of extreme values. Always remember to reverse the transform (exp(y) - 1) when interpreting predictions.

Target Encoding

Target encoding replaces each category with the mean of the target variable for that category. It is powerful for high-cardinality features (thousands of zip codes, for example) where one-hot encoding would create an unwieldy number of columns. However, it introduces a subtle but dangerous form of data leakage: the target variable is used to construct the feature. Rare categories are especially vulnerable—their target means are noisy and effectively memorize the training labels. Mitigate this with smoothing (blending the category mean with the global mean) and always compute target encoding within cross-validation folds.

Key Insight

In interviews and in practice, feature engineering often matters more than model choice. A simple logistic regression on well-crafted features can outperform a gradient boosting model on raw features. Invest time in understanding your data deeply—domain knowledge, not algorithmic complexity, is the moat.

Section 3.6

Cross-Validation and Data Leakage

The goal of validation is to estimate how well your model will perform on unseen data. Get this wrong, and you will ship a model that looks great in development and fails in production.

K-Fold Cross-Validation

In k-fold CV, you split the data into k equal parts (folds). For each of the k iterations, one fold serves as the test set and the remaining k − 1 folds form the training set. You train and evaluate k times, then average the scores. This gives you a more reliable estimate than a single train/test split because every data point appears in both training and testing exactly once.

Conceptually, picture a dataset split into five colored bands. In round one, the first band is held out; in round two, the second band; and so on. The final score is the mean across all five rounds. Five-fold or ten-fold CV is standard practice. 

from sklearn.model_selection import cross_val_score
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(n_estimators=200, random_state=42)
scores = cross_val_score(model, X, y, cv=5, scoring='f1')

print(f"F1 scores: {scores}")
print(f"Mean F1:   {scores.mean():.3f} (+/- {scores.std():.3f})")

Time Series Cross-Validation

Standard k-fold CV randomly shuffles the data, which is fatal for time series. If you train on data from March and test on data from January, your model has access to the future—a severe form of leakage. Time series CV respects temporal order: each training set includes only data before the test period, and the test window slides forward in time. Scikit-learn’s TimeSeriesSplit implements this pattern. 

from sklearn.model_selection import TimeSeriesSplit

tscv = TimeSeriesSplit(n_splits=5)
for train_idx, test_idx in tscv.split(X):
    X_train, X_test = X[train_idx], X[test_idx]
    y_train, y_test = y[train_idx], y[test_idx]
    # Train and evaluate — test always comes after train in time

The Cardinal Sin: Data Leakage

Data leakage occurs whenever information from outside the training set bleeds into the model during training. The most insidious form is fitting preprocessing on the full dataset before splitting. If you standardize features using the mean and standard deviation of the entire dataset (including the test set), the model implicitly “knows” something about the test data. The same applies to imputing missing values, computing target encodings, or selecting features based on correlations with the target.

The rule is simple: everything that learns from data must be fit inside the training fold only. Use Pipeline objects in scikit-learn to ensure preprocessing steps are properly contained within cross-validation. 

from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

# CORRECT: scaler is fit inside each CV fold
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('model',  LogisticRegression())
])
scores = cross_val_score(pipe, X, y, cv=5, scoring='accuracy')
Example 3.3 — How Leakage Inflates Accuracy

A data scientist builds a model to predict 30-day hospital readmission. During feature engineering, they compute each patient’s average length of stay, including the current visit (which hasn’t concluded at prediction time). They also scale all features using the mean and variance of the entire dataset before splitting into train and test sets.

The model reports 94% AUC on the test set. Excited, they deploy it. In production, the AUC drops to 68%. What happened?

Two leaks compounded. First, the average-length-of-stay feature encoded future information—you cannot know the length of a stay that has not ended. Second, standardizing on the full dataset meant the test set’s distribution was already “baked in” to the scaling parameters. Together, these leaks inflated the offline metric by over 25 points. The fix: compute only historically available features, and wrap all preprocessing inside a Pipeline within cross-validation.

Key Insight

If your model’s performance seems “too good to be true,” it almost certainly is. The most common culprit is data leakage. Before celebrating a high score, audit every feature: “Would I have this information at the moment of prediction?” If the answer is no, remove it.
