3.1 Supervised Learning Algorithms

▾

Supervised learning uses labeled data to learn a mapping from inputs to outputs. Each algorithm has its own geometric intuition, assumptions, and failure modes.

AlgorithmStrengthsWeaknessesWhen to Use
Linear RegressionInterpretable, fast, coefficients reveal importanceAssumes linearity, sensitive to outliersBaseline regression; interpretability needed
Logistic RegressionProbabilistic output, interpretableLinear decision boundary onlyBinary classification with explainability
Decision TreeNonlinear, no scaling, highly interpretableHigh variance, prone to overfittingQuick exploration; visual decision path
Random ForestLow variance, robust, handles missing valuesLess interpretable, slowerGeneral-purpose; accuracy without tuning
Gradient BoostingState-of-the-art tabular, flexible lossOverfitting risk, slow trainingCompetitions and production accuracy
SVMHigh-dim spaces, kernel flexibilitySlow, opaque, hyperparameter-sensitiveText classification; small/medium data
KNNNo training, complex boundariesSlow inference, curse of dimensionalitySmall datasets; anomaly detection

3.2 Unsupervised Learning

▾

Unsupervised learning discovers structure without labels. K-Means iterates between assigning points to nearest centroids and recomputing centroids. DBSCAN finds arbitrary-shaped clusters as dense regions separated by low-density areas. PCA finds axes of maximum variance for dimensionality reduction.

AlgorithmTypeKey PropertyUse When
K-MeansClusteringFast, spherical clustersKnown k, convex clusters
HierarchicalClusteringDendrogram explorationUnknown k, want hierarchy
DBSCANClusteringArbitrary shapes, finds outliersNo k needed, noise expected
PCADim. ReductionLinear, fast, deterministicFeature reduction, preprocessing
t-SNE / UMAPDim. ReductionNonlinear, preserves local structure2D visualization only
Worked Example — Customer Segmentation

After standardizing features (AOV, purchase frequency, recency), run K-Means with k=2 to 10. Elbow at k=4 reveals: Loyal High-Spenders, Bargain Hunters, Window Shoppers, and Churning Customers. Marketing tailors campaigns to each segment.

▾
MetricIdeaFails WhenUse When
Accuracy(TP+TN) / totalImbalanced classesBalanced datasets
PrecisionTP / (TP+FP)—Cost of FP is high (spam filter)
RecallTP / (TP+FN)—Cost of FN is high (disease detection)
F1 ScoreHarmonic mean of P & R—Need balance of precision and recall
AUC-ROCArea under TPR vs FPR—Comparing models, threshold-independent
RMSE√(mean squared errors)—Penalize large errors (regression)

Key Insight

Before choosing a metric, always ask: "What is the cost of a false positive versus a false negative?" The answer should drive your metric, threshold, model, and how you present results.

3.4 Bias-Variance Tradeoff

▾

The Dartboard Analogy: High bias = darts cluster tightly but away from bullseye (underfitting). High variance = darts scattered around bullseye (overfitting). Total Error = Bias² + Variance + Irreducible Noise.

ScenarioBiasVarianceSymptomFix
UnderfittingHighLowPoor on train AND testMore features, complex model
Sweet spotLowLowGood on bothKeep it
OverfittingLowHighGreat train, poor testRegularization, more data

L1 (Lasso) adds |coefficients| to loss — drives some to zero (feature selection). L2 (Ridge) adds coefficients² — shrinks all toward zero (stability). Elastic Net blends both.

3.6 Cross-Validation & Data Leakage

▾

K-Fold CV: Split data into k folds. Each iteration: 1 fold test, k−1 train. Every point appears in both roles. Time Series CV: Standard k-fold is fatal for time series — it trains on future data. Use expanding window that respects temporal order.

Data Leakage: The most insidious form is fitting preprocessing on the full dataset before splitting. If you standardize using the full dataset's mean/std, the model implicitly "knows" the test data. Rule: Everything that learns from data must be fit inside the training fold only. Use Pipeline.

from sklearn.model_selection import cross_val_score
scores = cross_val_score(model, X, y, cv=5, scoring='f1')
print(f"Mean F1: {scores.mean():.3f} (+/- {scores.std():.3f})")
Key Insight

If performance seems "too good to be true," it almost certainly is. The most common culprit is data leakage. Audit every feature: "Would I have this information at the moment of prediction?"
