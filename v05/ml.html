🤖 Supervised Learning

Learn the core algorithms that map inputs to labeled outputs.

📑 Core Algorithms

  • Linear Regression — fits a hyperplane minimizing MSE; assumes linearity and homoscedasticity
  • Logistic Regression — sigmoid output for binary classification; optimizes cross-entropy loss
  • Decision Trees — recursive splits on features using Gini impurity or information gain
  • Random Forest — ensemble of decorrelated trees via bagging + feature subsampling; reduces variance
  • XGBoost — sequential boosting with regularized objective; reduces bias iteratively
  • SVM — maximizes margin between classes; kernel trick for non-linear boundaries
  • KNN — classifies by majority vote of k nearest neighbors; lazy learner, sensitive to scale

⚖ Bias-Variance Tradeoff & Regularization

Total error = Bias² + Variance + Irreducible Noise. High bias → underfitting; high variance → overfitting.

  • L1 (Lasso) — adds |w| penalty; drives weights to exactly zero → sparse feature selection
  • L2 (Ridge) — adds w² penalty; shrinks weights toward zero but keeps all features
  • Elastic Net — combines L1 + L2 for correlated features

🔬 Unsupervised Learning

Discover patterns in unlabeled data through clustering and dimensionality reduction.

🌐 Clustering Algorithms

  • K-Means — partitions data into k clusters by minimizing within-cluster variance; sensitive to initialization (use k-means++)
  • Hierarchical — agglomerative (bottom-up) or divisive (top-down); produces a dendrogram; no need to pre-specify k
  • DBSCAN — density-based; finds arbitrarily shaped clusters; robust to outliers; needs eps and min_samples

📈 Dimensionality Reduction

  • PCA — linear projection onto principal components (eigenvectors of covariance matrix); preserves maximum variance
  • t-SNE — non-linear; preserves local neighborhood structure; great for visualization; not deterministic
  • UMAP — faster than t-SNE; better global structure preservation; works for high-dim embeddings

🔧 Feature Engineering

  • One-Hot Encoding — creates binary columns for categorical features; watch for high cardinality
  • Scaling — StandardScaler (z-score) or MinMaxScaler; critical for distance-based models
  • Log Transform — reduces right skew; stabilizes variance; use log1p for zeros
  • Target Encoding — replaces categories with mean of target; risk of leakage → use with CV folds

🎯 Model Evaluation

Measure model performance correctly and avoid common pitfalls.

📊 Classification Metrics

  • Accuracy — correct / total; misleading on imbalanced data
  • Precision — TP / (TP + FP); "of predicted positives, how many are correct?"
  • Recall — TP / (TP + FN); "of actual positives, how many did we catch?"
  • F1 Score — harmonic mean of precision and recall; balances both
  • AUC-ROC — area under ROC curve; threshold-independent ranking metric

📉 Regression Metrics

  • RMSE — root mean squared error; penalizes large errors more
  • MAE — mean absolute error; more robust to outliers than RMSE
  • R² — proportion of variance explained; 1.0 is perfect, can be negative for bad models

🔀 Cross-Validation & Leakage

  • k-Fold CV — splits data into k folds; each fold serves as validation once; standard is k=5 or 10
  • Stratified k-Fold — preserves class distribution in each fold; essential for imbalanced data
  • Time Series CV — expanding window; never use future data to predict the past
  • Data Leakage — occurs when info from outside training leaks in; e.g., fitting scaler on full data before splitting
