Query Basics

Query Execution Order

SQL doesn't execute top-to-bottom. The real order is:

FROM→ WHERE→ GROUP BY→ HAVING→ SELECT→ ORDER BY→ LIMIT

Key insight: You can't use a SELECT alias in WHERE because WHERE runs first.

Aggregation Gotchas

  • COUNT(*) — counts all rows, including NULLs
  • COUNT(col) — counts only non-NULL values in that column
  • WHERE — filters rows before grouping
  • HAVING — filters groups after aggregation
SELECT department, COUNT(*) AS cnt
FROM employees
WHERE active = 1        -- filters rows first
GROUP BY department
HAVING COUNT(*) > 5   -- filters groups after

JOIN Types

JOIN Types at a Glance

  • INNER JOIN — only matching rows from both tables
  • LEFT JOIN — all rows from left + matches from right (NULLs for non-matches)
  • RIGHT JOIN — all rows from right + matches from left
  • FULL OUTER JOIN — all rows from both, NULLs where no match
  • CROSS JOIN — cartesian product (every row × every row)
  • Self JOIN — join a table to itself (e.g., employee → manager)
-- LEFT JOIN: keep all users, even without orders
SELECT u.name, o.total
FROM users u
LEFT JOIN orders o ON u.id = o.user_id

Performance Tips

  • EXPLAIN / EXPLAIN ANALYZE — always check query plans
  • Index join columns — especially foreign keys
  • Avoid SELECT * — fetch only needed columns
  • Filter early — push WHERE conditions before joins when possible

Window Functions

Window Functions

Perform calculations across related rows without collapsing them.

  • ROW_NUMBER() — unique sequential number, no ties
  • RANK() — same rank for ties, skips next numbers
  • DENSE_RANK() — same rank for ties, no gaps
  • LAG(col, n) — value from n rows before
  • LEAD(col, n) — value from n rows ahead
SELECT
  name, department, salary,
  RANK() OVER (
    PARTITION BY department
    ORDER BY salary DESC
  ) AS dept_rank,
  LAG(salary, 1) OVER (
    ORDER BY salary DESC
  ) AS prev_salary
FROM employees

CTEs & Subqueries

Common Table Expressions make complex queries readable and reusable.

WITH dept_avg AS (
  SELECT department, AVG(salary) AS avg_sal
  FROM employees
  GROUP BY department
)
SELECT e.name, e.salary, d.avg_sal
FROM employees e
JOIN dept_avg d ON e.department = d.department
WHERE e.salary > d.avg_sal
  • CTEs — more readable than nested subqueries
  • Recursive CTEs — great for hierarchies and graphs
  • Subqueries — can appear in SELECT, FROM, or WHERE

What's the difference between RANK() and DENSE_RANK()?

Correct! RANK() leaves gaps (1,1,3) while DENSE_RANK() doesn't (1,1,2).
