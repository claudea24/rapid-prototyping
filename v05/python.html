Complexity Cheat Sheet

  • list — append O(1), index O(1), insert/delete O(n), search O(n)
  • dict — get/set/delete O(1) avg, worst O(n) due to hash collisions
  • set — add/remove/lookup O(1) avg, union O(m+n)
  • deque — appendleft/popleft O(1), index O(n)
  • heapq — push/pop O(log n), heapify O(n), peek O(1)

Common Coding Patterns

  • Two Pointers — sorted arrays, palindrome checks, partition
  • Hash Map — frequency count, two-sum, grouping by key
  • Sliding Window — max subarray, substring problems, running stats
# Two pointers: pair with target sum
def two_sum_sorted(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo < hi:
        s = arr[lo] + arr[hi]
        if s == target: return (lo, hi)
        elif s < target: lo += 1
        else: hi -= 1

# Sliding window: max sum of k elements
def max_sum_k(arr, k):
    w = sum(arr[:k])
    best = w
    for i in range(k, len(arr)):
        w += arr[i] - arr[i - k]
        best = max(best, w)
    return best

Python Gotchas

  • Mutable default args — def f(x=[]) shares the same list across calls!
  • Shallow copy — b = a[:] copies the list but nested objects still share references
  • Integer division — 7 // 2 = 3 (floor), -7 // 2 = -4 (rounds toward -inf)
  • is vs == — is checks identity (same object), == checks equality (same value)
# Mutable default gotcha
def bad(items=[]):
    items.append(1)
    return items

bad()   # [1]
bad()   # [1, 1]  ← surprise!

# Fix: use None sentinel
def good(items=None):
    items = items or []
    items.append(1)
    return items
