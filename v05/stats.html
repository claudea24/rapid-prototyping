Descriptive Stats

Locked

Core Measures

Summarize and describe the main features of a dataset:

  • Mean — arithmetic average of all values
  • Median — middle value when sorted; robust to outliers
  • Mode — most frequently occurring value
  • Variance — average of squared deviations from the mean
  • Standard Deviation — square root of variance; same units as data
  • IQR — interquartile range (Q3 − Q1); measures spread of the middle 50%

Confidence Intervals

A range of values likely to contain the true population parameter. A 95% CI means if we repeated the experiment many times, 95% of computed intervals would contain the true parameter.

CI = x̄ ± z* · (σ / √n)

The width depends on confidence level, variability (σ), and sample size (n). Larger samples yield narrower intervals.

💡 Quick Check

A p-value of 0.03 means...
  • There is a 3% probability of observing this data (or more extreme) if H0 is true
  • There is a 3% chance the null hypothesis is true
  • The treatment effect size is exactly 3%

Hypothesis Testing

Locked

Core Concepts

A framework for making statistical decisions using data:

  • H0 (Null Hypothesis) — the default assumption; no effect or difference
  • H1 (Alternative Hypothesis) — what we want to prove; there is an effect
  • p-value — probability of observing the data (or more extreme) if H0 is true
  • Type I Error — false positive; rejecting H0 when it is actually true (α)
  • Type II Error — false negative; failing to reject H0 when it is false (β)
  • Power — 1 − β; probability of correctly rejecting a false H0

Common Statistical Tests

Choose the right test for your data and hypothesis:

  • z-test — known population variance, large sample (n > 30)
  • t-test — unknown variance, small sample; compares means
  • Paired t-test — compares means from the same group at two different times
  • Chi-squared test — tests independence between categorical variables
  • ANOVA — compares means across 3+ groups simultaneously
  • Mann-Whitney U — non-parametric alternative to the t-test

💡 Quick Check

What is a Type II error?
  • Failing to reject a false null hypothesis (false negative)
  • Rejecting a true null hypothesis (false positive)
  • Choosing the wrong statistical test for the data

A/B Testing

Locked

5-Step Process

A structured approach to running controlled experiments:

  1. Define your metric and hypothesis clearly
  2. Calculate the required sample size for statistical power
  3. Randomly assign users to control and treatment groups
  4. Run the experiment for the pre-determined duration
  5. Analyze results and check for statistical significance

Common pitfalls: peeking at results early, multiple comparisons without correction, and novelty effects biasing short-term metrics.

Central Limit Theorem

Regardless of the population distribution, the distribution of sample means approaches a normal distribution as sample size increases (typically n ≥ 30). This underpins most A/B test analysis.

SE = σ / √n

The standard error (SE) decreases as sample size grows, meaning larger samples give more precise estimates of the population mean.

💡 Quick Check

If you peek at A/B test results daily and stop when significant, what happens?
  • The false positive rate inflates well above the nominal α level
  • Nothing changes; the p-value is still valid at any time
  • The test becomes more powerful due to continuous monitoring
