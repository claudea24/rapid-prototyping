Key Insight

Choosing the right data structure can turn an O(n²) solution into O(n). Checking for duplicates: O(n²) with nested loops, O(n) with a set. When a solution feels slow, ask: "Am I using the right container?"

Pandas: GroupBy & Merge

The groupby operation follows split-apply-combine: partition the DataFrame by keys, apply a function to each group, and stitch results back together. pd.merge() mirrors SQL joins — always verify row count after a merge to catch unexpected many-to-many duplicates.

import pandas as pd

summary = df.groupby('region')['revenue'].agg(
    total_revenue='sum',
    avg_order='mean',
    num_transactions='count'
)

# Merge: SQL-style joins in Pandas

# Two Sum with hash map: O(n)
def two_sum(nums, target):
    seen = {}
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
    return []
Key Insight

Most coding interview problems reduce to one of these four patterns. Before coding, identify the pattern first. "Is there a sorted structure I can exploit?" → two pointers. "Do I need fast lookups?" → hash map. "Contiguous subarray?" → sliding window.
