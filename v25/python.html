Key Insight: Checking duplicates with nested loops is O(n²). Pour into a set and compare lengths: O(n). The right container can change your complexity class.

# Two Sum — O(n) hash map approach
def two_sum(nums, target):
    seen = {}  # value -> index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
Key Insight: The hash map pattern is the single most important tool in coding interviews. Whenever a brute-force uses a nested loop where the inner loop is searching, replace it with a dictionary lookup.

# Mutable default — the bug
def append_to(item, target=[]):
    target.append(item)
    return target

print(append_to(1))   # [1]
print(append_to(2))   # [1, 2] — SAME list!

# The fix
def append_to(item, target=None):
    if target is None:
        target = []
    target.append(item)
    return target

Data Structures

  • list: O(1) index, O(1) append, O(n) search
  • dict/set: O(1) avg for all ops — use for lookup-heavy tasks
  • deque: O(1) both ends — BFS, sliding window
  • heapq: O(log n) push/pop — top-k problems

Pandas Essentials

  • groupby().agg() for split-apply-combine
  • merge(how='left') for SQL-style joins
  • Never loop over rows — vectorize or groupby

Coding Patterns

  • Hash map: replace inner search loop with dict lookup
  • Two pointers: sorted array pair problems, O(n)
  • Sliding window: subarray/substring, O(n)

Gotchas

  • Mutable defaults shared across calls — use None
  • Shallow copy shares inner objects — use deepcopy
  • is checks identity, == checks value
