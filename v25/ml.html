Exercise: Choose the Right Metric

⏱ 00:00

You are building a cancer screening model. Should you optimize for precision, recall, or accuracy? Why?

Recall. Missing a cancer case (false negative) is far more costly than a false alarm (false positive). Optimizing for recall ensures we catch as many actual cases as possible, even at the cost of some false positives that can be resolved with follow-up tests.

🎯 Bias-Variance Tradeoff FOCUS

▼

Total Error = Bias² + Variance + Irreducible Noise

ScenarioBiasVarianceSymptomFix
UnderfittingHighLowPoor on train AND testMore features, complex model
Sweet spotLowLowGood on bothKeep it
OverfittingLowHighGreat train, poor testRegularization, more data

L1 (Lasso): Drives coefficients to zero (feature selection). L2 (Ridge): Shrinks all coefficients (stability). Elastic Net: Both.

Dartboard Analogy: High bias = darts clustered away from bullseye. High variance = darts scattered widely. Sweet spot = tight cluster on bullseye.

🔓 Cross-Validation & Data Leakage FOCUS

▼

K-Fold CV: Split into k folds, rotate test fold. Every point appears in both roles. 5- or 10-fold is standard.

Time Series CV: Standard k-fold is fatal for time series (trains on future). Use expanding window that respects temporal order.

Data Leakage: The model sees info it would not have at prediction time. Most common: fitting preprocessing on full data before splitting.

from sklearn.model_selection import cross_val_score
from sklearn.pipeline import Pipeline

# Pipeline ensures scaler fits inside each fold
pipe = Pipeline([('scaler', StandardScaler()), ('model', LogisticRegression())])
scores = cross_val_score(pipe, X, y, cv=5, scoring='f1')
print(f"Mean F1: {scores.mean():.3f}")
Key Insight: If performance seems "too good to be true," it almost certainly is. The most common culprit is data leakage. Audit every feature: "Would I have this at prediction time?"

Algorithm Selection

  • Start simple: logistic/linear regression (baseline)
  • Random Forest: robust, handles nonlinearity, little tuning
  • Gradient Boosting (XGBoost/LightGBM): best tabular accuracy
  • Feature engineering often matters more than model choice

Metrics Quick Reference

  • Precision: Of flagged, how many correct? (spam filter)
  • Recall: Of actual positives, how many caught? (cancer screening)
  • F1: Harmonic mean — punishes extreme imbalance
  • AUC-ROC: Threshold-independent comparison

Bias-Variance

  • Underfitting: high bias, low variance (too simple)
  • Overfitting: low bias, high variance (too complex)
  • L1 = sparsity (feature selection), L2 = stability

Data Leakage

  • Fit all preprocessing inside CV folds (use Pipeline)
  • Time series: never train on future data
  • Ask: "Would I have this feature at prediction time?"
