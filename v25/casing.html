▼

North Star Metric: A single number capturing core value. Must align user and business success.

Key Insight: A good North Star aligns user value with business value. If it grows while users are unhappy, it is the wrong metric.

🔍 Diagnosing Metric Changes FOCUS

▼

The 6-Step Framework:

  1. Clarify — Restate the problem. Ask scope questions.
  2. Define Metrics — Primary metric + guardrails.
  3. Hypothesize — State hypotheses before touching data.
  4. Analyze — Segment by cohort, platform, geo, time.
  5. Recommend — Concrete action + expected impact.
  6. Measure — How will you validate? What experiment?

Investigation order: (1) Data pipeline issue? (2) External factor? (3) Internal change? (4) Segment to isolate. (5) Quantify contribution.

// Example: Revenue up 20% but rides down 5%
Revenue = Rides x Revenue_per_Ride
1.20 = 0.95 x X  →  X ≈ 1.263
Revenue per ride up ~26%

Hypotheses: price increase, mix shift to
longer rides, new premium tier launch.
Key Insight: Rule out data pipeline issues first — they are embarrassingly common and waste investigation time if not caught early.

Exercise: Metric Drop Investigation

⏱ 00:00

DAU dropped 10% week-over-week for a social media app. Walk through your investigation framework.

1. Clarify: Is this all platforms? Global? Compared to prior week or seasonal baseline? 2. Metrics: Primary = DAU. Guardrails = session length, revenue per user. 3. Hypothesize: (a) Data logging issue, (b) App crash on specific platform/version, (c) Holiday/external event, (d) Marketing campaign ended. 4. Analyze: Segment by platform → if iOS-only, check app version → crash logs spike in v5.2. Segment by geo → if global, less likely external. 5. Recommend: "Drop caused by iOS v5.2 crash. Hotfix + re-engagement push to affected users. Estimate recovery of ~8 of 10 lost points within one week." 6. Measure: Track iOS DAU daily. Monitor crash rate, App Store rating as leading indicators.

💰 Market Sizing FOCUS

▼
ApproachStart WithStructureStrength
Top-DownLarge known numberTAM → filter → targetFast, macro-level
Bottom-UpSingle unitUnit economics → scaleGrounded, concrete

Piano Tuners in Chicago (bottom-up):

  • 3.5M households * 5% piano = 175K + 10K institutional = 185K pianos
  • 1 tuning/year = 185K tunings
  • Tuner capacity: 4 tunings/day * 250 days = 1,000/year
  • 185K / 1K = ~185 tuners (actual: 100-300)
Key Insight: Interviewers care about structure and explicit assumptions, not the exact number. A clearly reasoned estimate off by 2x beats a lucky guess with no justification.

⚖ Experiment Design FOCUS

▼
DecisionDefaultWatch Out
Randomization unitUser-levelNetwork effects → cluster
Duration1+ full weekly cycleDay-of-week effects
GuardrailsAlways include severalMust not degrade
Novelty effectsRun 2-4 weeksInitial lift may fade

Example — New Checkout Flow: Primary metric = completion rate. Guardrails = AOV, return rate, support contacts. Power analysis: detect 2pp lift at alpha=0.05, power=0.80. Run 14 days minimum.

Communication tip: Lead with the answer (Pyramid Principle). State conclusion first, then evidence. Quantify everything. Acknowledge trade-offs proactively.

AARRR Framework

  • Acquisition → Activation → Retention → Revenue → Referral
  • Retention is most important (compounds)
  • North Star: aligns user value + business value

Metric Investigation

  • Clarify → Define Metrics → Hypothesize → Analyze → Recommend → Measure
  • Check data pipeline first
  • Segment by platform, geo, cohort, version

Market Sizing

  • Top-down: large number → filters
  • Bottom-up: single unit → scale
  • State assumptions explicitly, flag sensitivity

Experiment Design

  • User-level randomization (default)
  • Always set guardrail metrics
  • Run 2+ weeks, no peeking
  • Lead with the answer (Pyramid Principle)
