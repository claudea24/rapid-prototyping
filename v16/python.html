Scene 1 Choosing Your Tools — Data Structures

▾
You open the first file: 2.3 million rows of user IDs, but it's full of duplicates. Before reaching for Pandas, you think: what's the fastest way to deduplicate? The answer lies in choosing the right data structure.

Choosing the right data structure is often the difference between a solution that passes and one that times out. Checking whether a list contains duplicates takes O(n²) with nested loops, but only O(n) if you pour the elements into a set and compare lengths.

Choosing the right data structure can turn an O(n²) solution into O(n). When an interview solution feels too slow, the first question to ask is: "Am I using the right container?"

You use a set to identify 340,000 duplicate user IDs in seconds. Now you need to merge the three vendor files and clean the actual data. Time for Pandas.

Scene 2 Cleaning the Mess — Pandas

▾
The three vendor files have different column names, missing emails, and revenue stored as strings with dollar signs. You need groupby for segmentation, merge to combine files, and fillna for the gaps. And whatever you do, don't write a for-loop over rows.

The golden rule of Pandas: if you are writing a for-loop over rows, you are almost certainly doing it wrong. Use vectorized operations or groupby + agg.

import pandas as pd

df = pd.DataFrame({
    'region':  ['East', 'West', 'East', 'West', 'East'],
    'product': ['Widget', 'Gadget', 'Gadget', 'Widget', 'Widget'],
    'revenue': [120, 340, 210, 150, 95]
})

summary = df.groupby('region')['revenue'].agg(
    total_revenue='sum',
    avg_order='mean',
    num_transactions='count'
)
print(summary)
Key Insight

Always verify the resulting row count after a merge — unexpected duplicates from many-to-many joins are a common source of bugs. If your merged DataFrame is larger than expected, check for duplicate keys.

The three files are merged, cleaned, and segmented. But marketing also needs a deduplication script that runs on the raw ID lists. Your colleague suggests a hash map approach — it's a classic coding pattern.

Scene 3 Coding the Solution — Hash Map Pattern

▾
You need to match user IDs across systems. For each ID in system A, find the matching ID in system B. A nested loop would take hours on 2.3M records. The hash map pattern does it in one pass.

The hash map pattern uses a dictionary to remember previously seen values, converting nested-loop lookups into O(1) checks. It's the most versatile pattern in coding interviews.

# Naive approach: O(n^2)
def two_sum_naive(nums, target):
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]

# Hash map approach: O(n)
def two_sum(nums, target):
    seen = {}  # value -> index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i

print(two_sum([2, 7, 11, 15], 9))  # [0, 1]

Key Insight

Whenever a brute-force solution uses a nested loop where the inner loop is searching for something, ask: "Can I replace that search with a dictionary lookup?" If yes, you've just shaved an order of magnitude off your complexity.
