Performance Optimization

⚡ Key Optimization Strategies

  • Indexing: Create indexes on frequently queried columns
  • Query Structure: Use WHERE before JOIN, avoid SELECT *
  • Subqueries vs JOINs: JOINs often perform better than subqueries
  • EXPLAIN: Always analyze query execution plans

Example: Optimized Query

-- Before: Slow subquery approach SELECT e.name, e.salary, (SELECT AVG(salary) FROM employees WHERE department = e.department) as dept_avg FROM employees e WHERE e.salary > 100000; -- After: Optimized with window function SELECT name, salary, AVG(salary) OVER (PARTITION BY department) as dept_avg FROM employees WHERE salary > 100000;

Practice Scenarios

Scenario: E-commerce Analysis

You have orders, customers, and products tables. Find top 3 products by revenue in each category for Q4 2023.

WITH quarterly_sales AS ( SELECT p.category, p.product_name, SUM(o.quantity * o.unit_price) as revenue, ROW_NUMBER() OVER (PARTITION BY p.category ORDER BY SUM(o.quantity * o.unit_price) DESC) as rn FROM orders o JOIN products p ON o.product_id = p.id WHERE o.order_date BETWEEN '2023-10-01' AND '2023-12-31' GROUP BY p.category, p.product_name ) SELECT category, product_name, revenue FROM quarterly_sales WHERE rn <= 3 ORDER BY category, revenue DESC;
