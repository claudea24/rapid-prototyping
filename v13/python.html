Python — The Analyst's Toolkit

Python is the lingua franca of data science. This chapter covers not just syntax, but the reasoning behind choosing the right tool for each job — from data structures to Pandas patterns to common coding idioms.

2.1

Data Structures & Complexity

▼

Every data structure is a trade-off. A list gives you order and O(1) index access, but searching it is O(n). A dict gives you O(1) lookup by key, but consumes more memory. A set is a dict without values — perfect when you only care about membership. Understanding these trade-offs is the difference between code that works and code that scales.

The right mental model: before reaching for a data structure, ask yourself two questions. (1) What is my most frequent operation? (2) Does order matter? The answers will point you to the right container every time.

Key Insight

Choosing the right container turns O(n²) into O(n). When a solution feels slow, ask: am I using the right data structure?

Pandas Essentials

▼

The core philosophy of Pandas is split-apply-combine: split your data into groups, apply a function to each group, and combine the results back into a single structure. Once you internalize this pattern, most Pandas operations become intuitive.

The groupby operation is the canonical example. df.groupby('dept').agg({'salary': 'mean'}) splits the DataFrame by department, applies the mean function to each group's salary, and combines the results. Merge, pivot, and apply are all variations of this same idea.

Key Insight

If you're writing a for-loop over DataFrame rows, you're almost certainly doing it wrong. Pandas is designed for vectorized operations — use groupby, apply, or broadcasting instead.

2.3

NumPy Patterns

▼

Vectorization is the most important concept in NumPy. Instead of looping over elements one at a time (slow, Python-speed), you express operations on entire arrays at once (fast, C-speed). A simple example: a * 2 multiplies every element in array a by 2, without any loop.

The rule of thumb: if you're writing a Python loop over array elements, you're leaving performance on the table. NumPy's broadcasting rules let you combine arrays of different shapes without explicit iteration. Master broadcasting and you'll write code that's both faster and more readable.

import numpy as np

# Vectorized: fast (C-speed)
a = np.array([1, 2, 3, 4, 5])
result = a ** 2 + 2 * a + 1   # [4, 9, 16, 25, 36]

# Broadcasting: different shapes combine
matrix = np.array([[1,2,3], [4,5,6]])  # shape (2,3)
row    = np.array([10, 20, 30])            # shape (3,)
result = matrix + row  # broadcasts to (2,3)

# Boolean indexing: filter without loops
mask = a > 2
filtered = a[mask]  # array([3, 4, 5])

▼

Most coding interview problems are variations of a small set of patterns. Recognizing which pattern applies is more than half the battle. The table below covers the five you'll encounter most often in data science interviews.

PatternWhen to UseComplexity
Two PointersSorted array problemsO(n)

Counter / defaultdictFrequency countsO(n)

Worked example — Two Sum: Given an array and a target, find two numbers that add up to the target.

The brute force approach checks every pair: two nested loops, O(n²). The hash map approach stores each number's complement as you scan: one pass, O(n). For each number x, check if target - x is already in the map. If yes, return the pair. If not, store x.

# Brute Force: O(n²)
def two_sum_brute(nums, target):
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]

# Hash Map: O(n)
def two_sum(nums, target):
    seen = {}
    for i, x in enumerate(nums):
        complement = target - x
        if complement in seen:
            return [seen[complement], i]
        seen[x] = i
Key Insight

The hash map pattern is the single most important tool. It turns "have I seen this before?" from an O(n) scan into an O(1) lookup. When you're stuck, ask: can I trade space for time with a hash map?

Python Gotchas

▼

Python's simplicity hides a few sharp edges that trip up even experienced developers. These gotchas come up frequently in interviews — not as trick questions, but as tests of whether you truly understand the language's semantics.

GotchaWhat HappensThe Fix

# Gotcha: mutable default argument
def append_to(val, lst=[]):
    lst.append(val)
    return lst

print(append_to(1))  # [1]
print(append_to(2))  # [1, 2] — surprise!

# Fix: use None sentinel
def append_to_fixed(val, lst=None):
    if lst is None:
        lst = []
    lst.append(val)
    return lst
