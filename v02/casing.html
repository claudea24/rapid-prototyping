Case Studies

Product metrics, structured frameworks, market sizing, and experiment design for DS interviews.

Product Metrics Framework

Key
  • North Star Metric — single metric capturing core value
  • Acquisition — new users, sign-ups, installs
  • Activation — first key action (aha moment)
  • Retention — D1, D7, D30 retention rates
  • Revenue — ARPU, LTV, conversion rate
  • Referral — viral coefficient, NPS

Structuring a Case

Tip
  1. Clarify — restate the problem, ask questions
  2. Define metrics — primary + guardrail metrics
  3. Hypothesize — state expected cause / direction
  4. Analyze — segment data, test hypotheses
  5. Recommend — action + expected impact
  6. Measure — how to validate the recommendation

Metric Change Questions

Key

"Metric X dropped 10%. What happened?"

  1. Clarify — which metric, how measured, timeframe?
  2. External factors — seasonality, competitors, market
  3. Internal factors — product changes, bugs, deploys
  4. Segment — by platform, geo, user cohort, channel
  5. Quantify — how much does each factor explain?

Market Sizing

Tip
  • Top-down — Total market → apply filter rates
  • Bottom-up — Unit economics → scale up
  • State assumptions clearly
  • Use round numbers for mental math
  • Sanity-check your answer

Experiment Design

Key
  • Unit of randomization — user, session, page?
  • Duration — cover full weekly cycle minimum
  • Guardrail metrics — ensure you don't break things
  • Network effects — may need cluster randomization
  • Novelty / primacy effects — wait for stabilization

Communication Tips

Tip
  • Lead with the answer — conclusion first, then evidence
  • Structure your thoughts — "I'll break this into 3 parts..."
  • Quantify everything — "~20% of users" not "a lot of users"
  • Acknowledge trade-offs — shows maturity
  • Ask clarifying questions — never assume
