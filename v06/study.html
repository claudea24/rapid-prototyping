Query Execution Order

SQL doesn't execute in the order you write it. The actual processing order is:

  1. FROM & JOINs — tables are assembled
  2. WHERE — rows filtered
  3. GROUP BY — rows grouped
  4. HAVING — groups filtered
  5. SELECT — columns & expressions computed
  6. DISTINCT — duplicates removed
  7. ORDER BY — results sorted
  8. LIMIT / OFFSET — rows trimmed

This is why you can't use a column alias in WHERE but can in ORDER BY.

Key

JOIN Types

  • INNER JOIN — only matching rows from both tables
  • LEFT JOIN — all rows from left + matches from right (NULLs where no match)
  • RIGHT JOIN — all rows from right + matches from left
  • FULL OUTER JOIN — all rows from both; NULLs on non-matching side
  • CROSS JOIN — cartesian product (every combo)
  • SELF JOIN — table joined to itself (e.g., employee → manager)

Tip: LEFT JOIN + WHERE right.key IS NULL = rows in left not in right (anti-join).

KeyTip

Window Functions

Perform calculations across a set of rows related to the current row, without collapsing groups.

SELECT employee_id, department, salary, RANK() OVER ( PARTITION BY department ORDER BY salary DESC ) AS dept_rank, AVG(salary) OVER ( PARTITION BY department ) AS dept_avg FROM employees;

Key functions: ROW_NUMBER, RANK, DENSE_RANK, NTILE, LAG, LEAD, SUM, AVG.

Watch

Aggregation Gotchas

  • COUNT(*) counts all rows; COUNT(col) ignores NULLs
  • AVG ignores NULLs — can skew results vs. treating NULLs as 0
  • Any column in SELECT with an aggregate must be in GROUP BY (or itself aggregated)
  • HAVING filters after grouping; WHERE filters before
  • NULL comparisons: NULL = NULL is not TRUE; use IS NULL
  • GROUP BY 1,2 is shorthand for position; handy but less readable
Key

CTEs & Subqueries

Common Table Expressions make complex queries readable and reusable within a query.

WITH monthly_revenue AS ( SELECT DATE_TRUNC('month', order_date) AS month, SUM(amount) AS revenue FROM orders GROUP BY 1 ) SELECT month, revenue, LAG(revenue) OVER (ORDER BY month) AS prev_month FROM monthly_revenue;

Recursive CTEs useful for hierarchies and sequences.

Tip

Performance Tips

  • Index columns used in WHERE, JOIN, ORDER BY
  • Use EXPLAIN / EXPLAIN ANALYZE to inspect query plans
  • Avoid SELECT * — fetch only needed columns
  • Filter early: push predicates into subqueries / CTEs
  • Prefer EXISTS over IN for correlated subqueries
  • Avoid functions on indexed columns in WHERE (breaks index usage)
  • Partition large tables by date or category
  • Use LIMIT during exploration to avoid full scans
Key

Data Structures Complexity

  • list — append O(1), index O(1), insert/delete O(n), search O(n)
  • dict — get/set/delete O(1) avg, O(n) worst
  • set — add/remove/lookup O(1) avg
  • deque — append/pop both ends O(1)
  • heapq — push/pop O(log n), peek O(1)
  • defaultdict — like dict but auto-initializes missing keys

Tip: Use sets for membership checks, dicts for counting, deques for queues.

Watch

Gotchas

  • Mutable default args: def f(x=[]): is shared across calls. Use None instead.
  • Shallow vs deep copy: list.copy() doesn't clone nested objects. Use copy.deepcopy().
  • Float precision: 0.1 + 0.2 != 0.3 — use math.isclose() or Decimal.
  • is vs ==: is checks identity, == checks value equality.
  • Chained assignment: a = b = [] makes both point to the same list.
  • Global vs local: Assigning to a variable inside a function makes it local; use global or nonlocal if needed.
Key

Supervised Algorithms

  • Linear Regression — continuous target, interpretable coefficients, assumes linearity
  • Logistic Regression — binary classification, outputs probabilities, fast baseline
  • Decision Trees — non-linear, interpretable, prone to overfitting
  • Random Forest — ensemble of trees, reduces variance, robust
  • Gradient Boosting (XGBoost/LightGBM) — sequential trees, top performer on tabular data
  • SVM — effective in high dimensions, kernel trick for non-linearity
  • KNN — instance-based, no training, slow at prediction time
Key

Unsupervised Algorithms

  • K-Means — partition into k clusters, sensitive to initialization & outliers
  • Hierarchical Clustering — builds dendrogram, no need to specify k upfront
  • DBSCAN — density-based, finds arbitrary shapes, handles noise
  • PCA — dimensionality reduction, captures maximum variance
  • t-SNE / UMAP — non-linear embedding for visualization
  • Association Rules — market basket analysis (support, confidence, lift)

Evaluation Metrics

Classification:

  • Accuracy — (TP+TN) / total; misleading when imbalanced
  • Precision — TP / (TP+FP); "of predicted positive, how many correct?"
  • Recall — TP / (TP+FN); "of actual positive, how many found?"
  • F1 — harmonic mean of precision & recall
  • AUC-ROC — area under TPR vs FPR curve; threshold-independent

Regression: MSE, RMSE, MAE, R-squared

Key

Bias-Variance Tradeoff

  • Bias — error from overly simple models (underfitting)
  • Variance — error from sensitivity to training data (overfitting)
  • Total Error = Bias² + Variance + Irreducible Error

High bias fixes: more features, complex model, less regularization.

High variance fixes: more data, fewer features, regularization (L1/L2), ensembles, dropout.

Signal: Big gap between train & test accuracy = high variance. Both low = high bias.

Tip

Feature Engineering

  • Missing values: impute with mean/median/mode or create an indicator column
  • Encoding categoricals: one-hot (low cardinality), target encoding (high cardinality), ordinal
  • Scaling: StandardScaler (z-score) for linear models; MinMaxScaler for neural nets
  • Binning: convert continuous to categorical (e.g., age groups)
  • Interaction features: multiply or combine existing features
  • Date features: day of week, month, is_weekend, time since event
  • Log transform: right-skewed distributions
Key

Cross-Validation

  • K-Fold CV — split data into k folds, train on k-1, test on 1, rotate
  • Stratified K-Fold — preserves class distribution in each fold
  • Leave-One-Out — k = n; expensive but low bias
  • Time Series Split — expanding window; never train on future data
  • Nested CV — outer loop for evaluation, inner loop for hyperparameter tuning

Watch: Always cross-validate on the training set. The test set is touched only once at the end.

Key

Descriptive Stats

  • Mean — average; sensitive to outliers
  • Median — middle value; robust to outliers
  • Mode — most frequent value
  • Variance — average squared deviation from mean
  • Standard Deviation — square root of variance; same units as data
  • Skewness — asymmetry (positive = right tail, negative = left tail)
  • Kurtosis — tail heaviness; high = more outliers
  • IQR — Q3 - Q1; used for outlier detection (1.5 × IQR rule)
Key

Hypothesis Testing

  1. State H₀ (null) and H₁ (alternative)
  2. Choose significance level α (typically 0.05)
  3. Select the appropriate test
  4. Compute the test statistic and p-value
  5. If p-value < α, reject H₀
  • Type I Error (α) — false positive; rejecting a true H₀
  • Type II Error (β) — false negative; failing to reject a false H₀
  • Power = 1 - β; probability of correctly rejecting H₀
Key

Common Tests

  • t-test (1-sample) — is the mean different from a known value?
  • t-test (2-sample) — do two groups have different means?
  • Paired t-test — before/after on the same subjects
  • Chi-squared test — association between categorical variables
  • ANOVA — compare means across 3+ groups
  • Mann-Whitney U — non-parametric alternative to 2-sample t-test
  • Kolmogorov-Smirnov — test if data follows a distribution
KeyTip

A/B Testing

  • Randomization — randomly assign users to control/treatment
  • Sample size — compute upfront based on MDE, α, power
  • Metric selection — primary metric (OEC) + guardrail metrics
  • Duration — run long enough to capture weekly cycles
  • Multiple testing — Bonferroni or FDR correction if many comparisons
  • Novelty & primacy effects — users may behave differently at first
  • Network effects — spillover can violate SUTVA; use cluster randomization

Confidence Intervals

A range of plausible values for a population parameter.

CI = x̄ ± z* · (σ / √n) For proportions: CI = p̂ ± z* · √(p̂(1-p̂) / n)

z* = 1.96 for 95% CI, 2.576 for 99% CI

Interpretation: "We are 95% confident the true parameter lies in this interval." It does not mean 95% probability the parameter is in the interval.

FormulaKey

Central Limit Theorem

If X₁, X₂, ..., Xₙ are i.i.d. with mean μ and variance σ², then: X̄ ~ N(μ, σ²/n) as n → ∞
  • Works regardless of the underlying distribution
  • Generally kicks in around n ≥ 30
  • Foundation for confidence intervals and hypothesis tests
  • Explains why the normal distribution appears so often in practice
Key

Fundamentals

  • P(A) is between 0 and 1
  • Complement: P(A') = 1 - P(A)
  • Addition rule: P(A∪B) = P(A) + P(B) - P(A∩B)
  • Multiplication rule: P(A∩B) = P(A) · P(B|A)
  • Independence: P(A∩B) = P(A) · P(B) iff A, B independent
  • Conditional: P(A|B) = P(A∩B) / P(B)
  • Law of Total Probability: P(A) = Σ P(A|Bᵢ) · P(Bᵢ)
FormulaKey

Bayes' Theorem

P(A|B) = P(B|A) · P(A) / P(B) Equivalently: Posterior = Likelihood × Prior / Evidence

Classic example: Disease testing — even a 99% accurate test has a high false positive rate if the disease prevalence is low.

Steps: 1) Define prior P(A), 2) Compute likelihood P(B|A), 3) Compute evidence P(B) using total probability, 4) Apply formula.

Key

Key Distributions

  • Bernoulli — single trial, success/failure
  • Binomial(n,p) — number of successes in n independent trials
  • Poisson(λ) — count of events in a fixed interval; λ = rate
  • Geometric(p) — number of trials until first success
  • Uniform(a,b) — all values equally likely
  • Normal(μ,σ²) — bell curve; 68-95-99.7 rule
  • Exponential(λ) — time between Poisson events; memoryless

Expected Value & Variance

E[X] = Σ xᵢ · P(xᵢ) Var(X) = E[X²] - (E[X])² E[aX + b] = a·E[X] + b Var(aX + b) = a²·Var(X)

Linearity of expectation always holds, even for dependent variables:

E[X + Y] = E[X] + E[Y]

Variance of sum (independent): Var(X+Y) = Var(X) + Var(Y)

Key

Law of Large Numbers

  • Weak LLN: Sample average converges in probability to the expected value as n → ∞
  • Strong LLN: Sample average converges almost surely to the expected value
  • Foundation for why averages stabilize with more data
  • Gambler's fallacy: Past outcomes don't affect future independent events; LLN applies to averages, not individual trials

Practical implication: Larger samples give more reliable estimates. But the rate of improvement is √n, not n — diminishing returns.

Classic Interview Puzzles

  • Birthday problem: ~23 people for 50% chance of a shared birthday (pigeonhole + complement counting)
  • Monty Hall: Always switch doors; 2/3 chance of winning
  • Coupon collector: E[n] = n · Hₙ to collect all n types (Hₙ = harmonic number)
  • Dice expected value: E[fair die] = 3.5; E[rolls to get a 6] = 6
  • Card drawing: P(ace then king) — consider with vs without replacement
  • Random walk: 1D symmetric walk returns to origin with probability 1
Key

Product Metrics Framework

  • Acquisition — how do users find us? (signups, installs, CTR)
  • Activation — do they have a good first experience? (onboarding completion, "aha" moment)
  • Engagement — how often do they return? (DAU/MAU, session length, actions per session)
  • Retention — do they stick around? (D1, D7, D30 retention)
  • Revenue — do they pay? (ARPU, LTV, conversion rate)
  • Referral — do they bring others? (NPS, viral coefficient)

Tip: Always define a North Star metric that aligns with long-term value.

Tip

Structuring a Case

  1. Clarify — restate the question, ask about context, users, and goals
  2. Define metrics — choose a primary metric and supporting metrics
  3. Explore the data — segment by user type, platform, geography, time
  4. Form hypotheses — list possible explanations; prioritize by likelihood and impact
  5. Validate — describe how you'd test each hypothesis with data
  6. Recommend — propose actions with expected tradeoffs

Key: Think out loud. Interviewers care about your process as much as the answer.

KeyWatch

Metric Change Questions

"X metric dropped 10% this week. What happened?"

  • 1. Verify — data issue? Logging bug? Definition change?
  • 2. Decompose — metric = numerator / denominator; which moved?
  • 3. Segment — by platform, region, user cohort, feature
  • 4. Timeline — when exactly? Correlate with releases, events, seasonality
  • 5. External factors — competitors, holidays, press coverage
  • 6. Prioritize — rank hypotheses, propose investigation plan
Tip

Market Sizing

  • Top-down: Start with total market, narrow by percentages (TAM → SAM → SOM)
  • Bottom-up: Start with unit economics, scale up
  • State assumptions explicitly and sanity-check them
  • Round numbers for easy mental math
  • Use anchors: US population ~330M, world ~8B, households ~130M
  • Show your reasoning clearly; precision matters less than logic

Example: "How many piano tuners in Chicago?" Break into: pianos → tuning frequency → time per tuning → tuners needed.

Key

Experiment Design

  • Unit of randomization — user, session, device, or cluster?
  • Metric selection — OEC (Overall Evaluation Criterion) + guardrails
  • Sample size calculation — based on baseline rate, MDE, α=0.05, power=0.80
  • Duration — capture full weekly cycles; typically 1-2 weeks minimum
  • Avoiding pitfalls: peeking (use sequential testing), Simpson's paradox (check segments), interference (use isolation)
  • Ramp-up plan — 1% → 10% → 50% → 100% to catch issues early
Tip

Communication Tips

  • Start with the answer — lead with the conclusion, then support with data
  • Use frameworks — show structured thinking (MECE, 2x2, funnel)
  • Quantify — "a lot of users" → "~30% of DAU"
  • Acknowledge tradeoffs — show you understand business context
  • State assumptions — be explicit about what you're assuming
  • Ask clarifying questions — don't dive in before understanding the problem
  • Summarize — end with a crisp recap of findings and recommendations
